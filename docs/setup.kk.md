# Баптау нұсқаулығы

## Қажетті шарттар

- **Телефон:** USB жөндеуі қосылған кез келген Android 10+ құрылғысы
- **Автокөлік:** BYD DiLink 3.0+ (немесе USB хост порты бар кез келген Android 10+ бас құрылғысы)
- **USB кабелі:** Телефоннан автокөліктің USB портына
- **Әзірлеу:** Android Studio немесе Gradle, JDK 17

## Телефонды баптау (бір реттік)

1. **DiLink Auto Client орнатыңыз**: `adb install app-client-debug.apk`
2. **Қолданбаны ашыңыз** — бастапқы баптау экраны сізді әр рұқсат бойынша бағыттайды:
   - **Барлық файлдарға қол жеткізу** — виртуалды дисплей серверін жадқа орналастырады
   - **Батареяны оңтайландыру** — экран өшірулі кезде трансляцияны сақтайды
   - **Арнайы мүмкіндіктер қызметі** — автокөліктен сенсорлық басқаруды қосады
   - **Хабарландыруларға қол жеткізу** — телефон хабарландыруларын автокөлік дисплейіне жібереді
3. Әр қадам тиісті жүйелік баптауларды ашады. Рұқсатты беріңіз, содан кейін жалғастыру үшін Артқа түймесін басыңыз.
4. Кез келген қадамды өткізіп жіберіп, оны кейін негізгі экраннан баптауға болады.

Болды. Сымсыз жөндеу, жұптау кодтары, арнайы WiFi баптауы қажет емес.

## Автокөлікті баптау

Автокөлікке қолмен орнату қажет емес. Автокөлік APK телефон APK ішіне ендірілген. Алғашқы қосылуда (немесе нұсқа сәйкессіздігінде) телефон автокөлікке `UPDATING_CAR` жібереді (ол "Жаңарту..." күйін көрсетеді), содан кейін dadb (WiFi ADB) арқылы автоматты түрде орнатады.

Балама ретінде телефон қолданбасындағы "Автокөлікке орнату" түймесін пайдаланып, автокөлік APK-сын қолмен жіберуге болады.

Қолмен орнату кезінде: `adb install app-server-debug.apk` (автокөлікке ADB қолжетімділігін қажет етеді).

## Күнделікті пайдалану

1. **Телефонды автокөліктің USB портына жалғаңыз** — автокөлік қолданбасы автоматты түрде іске қосылады (немесе бір желіде WiFi арқылы қосылыңыз)
2. Автокөлік пен телефон параллель WiFi + USB тректері арқылы автоматты түрде қосылады
3. Телефон VD серверін орналастырады және трансляция 60 кадр/с жылдамдықпен басталады
4. Телефон қолданбаларымен әрекеттесу үшін автокөліктің сенсорлық экранын пайдаланыңыз

## Құрастыру

```bash
# Телефон APK құрастыру (серверді құрастыру + ендіруді автоматты түрде іске қосады)
./gradlew :app-client:assembleDebug

# APK орналасуы:
# app-client/build/outputs/apk/debug/app-client-debug.apk  (телефон -- ендірілген автокөлік APK-сын қамтиды)
```

Құрастыру жүйесі app-server-ді автоматты түрде құрастырады және оны app-client-ке ендіреді, сондықтан тек бір APK-ны қолмен орнату қажет.

## Бұл қалай жұмыс істейді

Телефон автокөлікке қосылған кезде:

1. **Автокөлік параллель тректерді іске қосады** — WiFi табу және USB анықтау бір уақытта орындалады
2. **А трегі (WiFi):** шлюз IP + mDNS табу → телефонның басқару портына NIO қосылу (9637)
3. **Б трегі (USB):** құрылғыларды сканерлеу → USB ADB қосылу → `am start` арқылы телефон қолданбасын іске қосу
4. **Handshake:** автокөлік viewport + DPI + appVersionCode + targetFps жібереді; телефон құрылғы ақпараты + vdServerJarPath жібереді
5. **Нұсқаны тексеру:** телефон appVersionCode салыстырады — сәйкес келмесе, автокөлікке UPDATING_CAR хабарын жібереді, содан кейін dadb арқылы автоматты түрде жаңартады
6. **3 қосылымды баптау:** автокөлік handshake-тен кейін video (9638) + input (9639) қосылымдарын ашады
7. **Телефон VD серверін орналастырады** — vd-server.jar-ды `/sdcard/DiLinkAuto/` ішіне шығарады, `app_process`-ті shell UID ретінде FPS аргументімен іске қосады
8. **VD сервері телефонға кері қосылады** localhost:19637 бойынша (блокталмайтын NIO)
9. **VD сервері VirtualDisplay жасайды** телефонның табиғи DPI (480dpi) бойынша GPU кішірейтуімен және мерзімді қайта салумен
10. **Бейне WiFi TCP арқылы беріледі** (video қосылымы 9638) — H.264, Main профилі, 8 Мбит/с CBR, 60 кадр/с дейін бапталады

## Ақаулықтарды жою

### ADB аутентификация диалогы әр уақытта пайда болады (v0.13.1 ТҮЗЕТІЛДІ)
Түзетілді — мәселе AUTH_TOKEN-ды екі рет хэштеуде болды. ADB 20 байттық шикі токен жібереді, оны алдын ала хэштелген SHA-1 дайджесті ретінде қарастыру керек. Ескі код `SHA1withRSA` қолданды, ол оны қайта хэштеді. Енді AOSP `RSA_sign(NID_sha1)` сәйкес келетін SHA-1 DigestInfo префиксімен (алдын ала хэштелген) `NONEwithRSA` қолданылады. Телефон қайта қосылу кезінде AUTH_SIGNATURE қабылдайды және "Әрқашан рұқсат ету" дұрыс сақталады.

Егер диалог жаңартудан кейінгі алғашқы қосылуда әлі де пайда болса, "Әрқашан рұқсат ету" белгілеңіз — ол қайта пайда болмауы керек. Егер мәселе шешілмесе, Әзірлеуші опцияларында "ADB авторизация тайм-аутын өшіру" (Android 11+) тексеріңіз.

### Телефон қолданбасы іске қосылмайды
- Телефонда USB жөндеу қосылғанына көз жеткізіңіз
- Автокөліктің USB порты хост режимін қолдайтынын тексеріңіз (барлық порттар қолдамайды)
- Басқа USB кабелін қолданып көріңіз (кейбір кабельдер тек зарядтауға арналған)

### Барлық файлдарға қол жеткізу рұқсаты берілмеді
- Телефон қолданбасы vd-server.jar-ды `/sdcard/DiLinkAuto/` ішіне орналастыру үшін MANAGE_EXTERNAL_STORAGE қажет етеді
- Баптаулар -> Қолданбалар -> DiLink Auto -> Рұқсаттар -> Барлық файлдарға қол жеткізу -> ҚОСУ

### Бейне көрсетілмейді / қара экран
- Телефон мен автокөліктің бір желіде екеніне көз жеткізіңіз
- Екі қолданбаның да жұмыс істеп тұрғанын тексеріңіз (телефон "Трансляция" көрсетеді, автокөлік бейнені көрсетеді)
- VD серверіне іске қосылу үшін уақыт қажет болуы мүмкін — қосылғаннан кейін 5-10 секунд күтіңіз
- Мерзімді қайта салуы бар SurfaceScaler статикалық мазмұнда да кадрлар шығаруы керек
- Диагностикалық ақпарат үшін `/sdcard/DiLinkAuto/client.log` тексеріңіз

### Қосылым үзіледі
- Бұрын байланыссыз желі үзілістері (мобильді деректерді ауыстыру) проактивті ажыратуды тудырды
- v0.12.5 түзетілді: ақылды желі кері шақыруы қосылымды тасымалдамайтын желілердегі үзілістерді елемейді
- Егер мәселе шешілмесе, `/sdcard/DiLinkAuto/client.log` ішінен "Network lost" жазбаларын тексеріңіз

### Автокөлік қолданбасы жаңартылмайды
- Телефон handshake кезінде нұсқа сәйкессіздігі анықталғанда автокөлік қолданбасын автоматты түрде жаңартады
- Автокөлік жаңарту кезінде "Автокөлік қолданбасы жаңартылуда..." күйін көрсетеді
- Сондай-ақ телефон қолданбасындағы "Автокөлікке орнату" түймесімен жаңартуды қолмен іске қосуға болады
- dadb автокөлікке WiFi ADB арқылы қол жеткізе алатынына көз жеткізіңіз

### Логтар
- Телефон логтары: `/sdcard/DiLinkAuto/client.log` (ағымдағы сессия)
- Алдыңғы сессиялар: `/sdcard/DiLinkAuto/client-YYYYMMDD-HHmmss.log`
- VD серверінің логтары: `/data/local/tmp/vd-server.log` (телефонда, ADB арқылы оқылады)
- Автокөлік логтары: протокол DATA арнасы арқылы телефонның client.log ішіне бағытталады (тег: `CarLog`)
- Логтарды алу: `adb shell "cat /sdcard/DiLinkAuto/client.log"`

## HyperOS (Xiaomi) бойынша кеңестер

HyperOS жүйесінде сенімді жұмыс істеу үшін:
1. Баптаулар -> Қолданбалар -> DiLink Auto -> Автоіске қосу -> Қосу
2. Баптаулар -> Батарея -> DiLink Auto -> Шектеусіз
3. Қолданбаны соңғы қолданбаларда бекітіңіз (картаны ұзақ басу -> Бұғаттау)

## Samsung One UI бойынша кеңестер

One UI 5+ (Android 13+) жүйесіндегі Samsung құрылғыларында DiLink-Auto дұрыс жұмыс істеуіне кедергі келтіретін қосымша қауіпсіздік және энергия үнемдеу мүмкіндіктері бар. Бұл Galaxy A, M, S, Z және Tab серияларына қолданылады.

### Auto Blocker өшіру (USB ADB үшін өте маңызды)

**Auto Blocker** USB командаларын бұғаттайды және автокөліктің телефонға USB ADB арқылы қосылуына кедергі келтіруі мүмкін. Бұл Samsung-қа тән ең кең таралған мәселе.

1. **Баптаулар → Қауіпсіздік және құпиялылық → Auto Blocker → Өшірулі**
2. Auto Blocker қосулы күйінде қалдыруды қаласаңыз, кем дегенде **"USB кабелі арқылы командаларды бұғаттау"** опциясын өшіріңіз

### Барлық файлдарға қол жеткізуге рұқсат беру

Samsung рұқсаттар менеджері сіз жақында ашпаған қолданбалардың рұқсаттарын автоматты түрде қайтарып алуы мүмкін:

1. **Баптаулар → Қолданбалар → DiLink Auto → Рұқсаттар → Файлдар және медиа → Барлық файлдарды басқаруға рұқсат беру**
2. **"Барлық файлдарды басқаруға рұқсат беру"** тумблерін қосыңыз
3. Баптауларды жапқаннан кейін оның ҚОСУЛЫ күйінде қалғанын тексеріңіз (Samsung растау поп-апын көрсетуі мүмкін)

### Батареяны оңтайландыруды өшіру

Samsung батареяны басқаруы стандартты Android-қа қарағанда агрессивті:

1. **Баптаулар → Қолданбалар → DiLink Auto → Батарея → Шектеусіз**
2. **Баптаулар → Батарея → Фондық пайдалану шектеулері → Ешқашан ұйықтамайтын қолданбалар → DiLink Auto қосу**
3. **Баптаулар → Батарея → Фондық пайдалану шектеулері → Терең ұйқыдағы қолданбалар → DiLink Auto жою** егер тізімде болса

### Қолданбаны соңғыларда бекіту

Samsung One UI жадыны босату үшін фондық қолданбаларды өшіруі мүмкін:

1. Соңғы қолданбаларды ашыңыз (3 түймелі навигациямен төменнен жоғары сырғытыңыз немесе жест навигациясы)
2. Картаның жоғарғы жағындағы DiLink Auto белгішесін түртіңіз
3. **"Ашық қалдыру"** тармағын таңдаңыз

### Samsung Device Care автоматты оңтайландыруын өшіру

Samsung Device Care фондық қызметтерді автоматты түрде тоқтатуы мүмкін:

1. **Баптаулар → Батарея және құрылғы күтімі → Автоматтандыру → Күнделікті автооңтайландыру → Өшірулі**
2. **Баптаулар → Батарея және құрылғы күтімі → Автоматтандыру → Автоқайта қосу → Өшірулі**

### "DeX" рұқсат поп-апын көрсеңіз

Кейбір Samsung құрылғылары қолданба виртуалды дисплей жасауға әрекеттенгенде "Samsung DeX" немесе "сыртқы дисплей" рұқсаттары туралы поп-ап көрсетеді. Galaxy A/M сериялары DeX қолдамаса да, диалог пайда болуы мүмкін. Жай ғана **"Рұқсат беру"** немесе **"Қазір бастау"** түймесін басыңыз. Егер диалог қайталана берсе, **Баптаулар → Қосылған құрылғылар → Samsung DeX** бөліміне өтіп, "HDMI қосылғанда автоматты іске қосу" опциясын өшіріңіз.

### Knox қауіпсіздік ескертулері

Samsung Knox келесілерге қол жеткізген кезде қауіпсіздік хабарландыруын көрсетуі мүмкін:
- Виртуалды дисплей беті (бейнені кодтау үшін)
- USB жөндеу көпірі (автокөлік ADB қосылымы үшін)
- Барлық файлдар қоймасы (VD серверін орналастыру үшін)

Бұл күтілетін әрекеттер. Knox-қа қатысты кез келген сұрауларда "Рұқсат беру" немесе "OK" түймесін басыңыз. Егер сұраулар тоқтамаса, Knox қорғанысын уақытша "Орташа" деңгейіне төмендетуге болады: **Баптаулар → Қауіпсіздік және құпиялылық → Samsung Knox**.
