# DiLink-Auto

Телефоныңыздағы қолданбаларды көлігіңіздің кіріктірілген экранында пайдаланыңыз. Ашық бастапқы коды, Google қызметтері қажет емес.

**Android 10+ жүйесіндегі кез келген телефон** мен **BYD DiLink 3.0+** ақпараттық-ойын-сауық жүйелері үшін Android Auto-ға ашық бастапқы кодты балама. Бастапқыда Xiaomi HyperOS / Қытай ROM алшақтығынан туындады, бірақ әмбебап жұмыс істейді.

## Не Істейді

DiLink-Auto телефоныңыздағы қолданбаларды толық сенсорлық өзара әрекеттесу арқылы көлігіңіздің дисплейіне бейнелейді. Навигацияны, музыканы, хабарландыруларды — телефондағы кез келген қолданбаны — тікелей көлік экранынан іске қосыңыз. Хабарландырулар көліктің шарлау тақтасында прогресс көрсеткіштерімен бірге пайда болады. H.264 бейне 60 кадр/с дейін, 8 Мбит/с CBR, батареяны үнемдеу үшін телефон экраны өшірулі.

**Бастапқы мотивация:** телефоныңыз Android Auto жүйесін іске қоса алмаған кезде (Қытай ROM, Google Play Services жоқ), бірақ көлігіңіз тек Android Auto қолдайтын (CarWith, CarPlay немесе Carlife жоқ) алшақтықты жою. Бірақ DiLink-Auto кез келген Android телефонымен жұмыс істейді — Google қызметтері бар немесе жоқ.

| Құрылғы | Мәселе |
|--------|-------|
| Xiaomi 17 Pro Max (HyperOS 3, Қытай ROM) | Android Auto жоқ — Google Play Services жоқ |
| BYD Destroyer 05 / King (Бразилия нарығы) | Бас құрылғыда тек Android Auto |
| Android 10+ кез келген телефон | ROM немесе Play Services қарамастан жұмыс істейді |

## Талаптар

**Телефон:**
- Android 10+ кез келген телефон
- USB жөндеу қосылған (Әзірлеуші параметрлері)
- Барлық файлдарға қол жеткізу рұқсаты (алғашқы іске қосуда сұралады)

**Көлік:**
- BYD DiLink 3.0 немесе одан жаңа
- Бір бос USB-A порты

**Телефон хотспоты қосылуы керек** — көлік телефоныңыздың WiFi хотспотына қосылады. Жұптастыру кодтары қажет емес, Google аккаунты қажет емес.

**Интернетке қосылу қажет емес.** DiLink-Auto барлығын телефоныңыздың WiFi кіру нүктесі арқылы жергілікті түрде таратады — көлік пен телефон тікелей байланысады. Интернетке қосылу тек телефоныңызда жұмыс істейтін қолданбалар үшін қажет (мысалы, навигация, музыка), бірақ DiLink-Auto үшін емес.

## Қалай Жұмыс Істейді

1. **Хотспотты қосыңыз** — Телефоныңыздың WiFi хотспотын қосыңыз. Көлік оған қосылады.
2. **Жалғаңыз** — Телефонды көліктің USB портына қосыңыз
3. **Авто-орнату** — Телефон WiFi ADB арқылы көлік қолданбасын орнатады (тек бірінші рет, бір рет түрту)
4. **Авто-қосылу** — 3 арнайы WiFi TCP ағыны: бейне (9638 порт), сенсорлық енгізу (9639 порт) және басқару (9637 порт)
5. **Қолданбаларыңызды пайдаланыңыз** — Көліктің іске қосу экранынан кез келген қолданбаны іске қосыңыз. Ол телефонда жұмыс істейді, көлікте көрінеді және түртуге жауап береді

Телефон қолданбаларыңызды виртуалды дисплейде іске қосады, экранды H.264 бейне ретінде кодтайды және оны көлікке ағынмен жібереді. Көлік экранындағы түртулер телефонға кері жіберіледі және нақты сенсорлық оқиғалар ретінде енгізіледі. Телефонның физикалық экраны өшірулі қалады (батареяны үнемдеу) және тәуелсіз пайдаланылуы мүмкін.

## Орнату

<a href="https://github.com/andersonlucasg3/dilink-auto-android/releases/latest"><img src="https://img.shields.io/github/v/release/andersonlucasg3/dilink-auto-android?label=Download%20Latest%20Release" alt="Download Latest Release"></a>

Соңғы шығарылымды жүктеп алыңыз немесе бастапқы кодтан құрастырыңыз:

1. **Құрастыру:** `./gradlew :app-client:assembleDebug`
2. `app-client/build/outputs/apk/debug/app-client-debug.apk` жолындағы APK-ны **тек телефоныңызға орнатыңыз**
3. Телефоныңызда **USB жөндеуді қосыңыз** (Параметрлер → Әзірлеуші параметрлері)
4. Телефонда **DiLink-Auto** қолданбасын ашып, сұралған кезде Барлық файлдарға қол жеткізу рұқсатын беріңіз
5. **Хотспотты қосыңыз, содан кейін көлік USB-ге жалғаңыз** — көлік қолданбасы WiFi ADB арқылы бірінші іске қосуда авто-орнатылады

Көлік APK және VD сервер JAR телефон APK ішіне ендірілген — көлікке ештеңені өзіңіз орнатудың қажеті жоқ.

## Ағымдағы Күй

**Жұмыс істейді:**
- 60 кадр/с H.264 бейне ағыны (8 Мбит/с CBR, Main профилі, келісу арқылы бапталады)
- Толық сенсорлық енгізу (мультитач, масштабтау үшін шымшу)
- Іздеу, алфавиттік сұрыптау, 64dp белгішелері бар қолданба іске қосқышы
- Прогресс жолақтары бар көлік экранындағы хабарландырулар, ашу үшін түртіңіз
- GitHub Releases (шығарылым) немесе алдын ала шығарылымдар (жөндеу) арқылы өзін-өзі жаңарту
- Авто-жаңарту: телефон ескірген көлік қолданбасын анықтайды және оны WiFi ADB арқылы жаңартады
- Ағын кезінде телефон экраны өшірулі (батареяны үнемдеу)
- Барлық қажетті рұқсаттар үшін бағытталған кіріспе
- Интернационалдандыру: ағылшын, португал, орыс, беларусь, француз, қазақ, украин, өзбек
- USB ажыратылғаннан кейін дисплейді қалпына келтіру (v0.14.0+)
- BYD DiLink 3.0 (1920x990) + Xiaomi 17 Pro Max (Android 16) + POCO F5 жүйелерінде тексерілген

**Жуырда:** аудио ағыны, медиа басқару, навигация виджеттері

**Белгілі шектеулер:**
- VD сервер процесі USB ажыратылғанда қайта іске қосылады (автоматты түрде қайта қосылады).
- Хотспот қолмен қосылуы керек (Android 16 шектеуі).
- Кейде визуалды артефактілер — декодерді қайта іске қосу жарысы, келесі кілттік кадрда қалпына келеді (~1 с).
- Жүктеме кезінде ағын кідірісі ~100-200 мс. CBR 8 Мбит/с.
- Кенеттен USB ажыратылғаннан кейін дисплей өшірулі қалуы мүмкін (v0.14.0 нұсқасында түзетілген).

## Жоба Құрылымы

Телефон APK (`app-client`) көлік APK (`app-server`) және VD сервер JAR екеуін де қамтиды. Телефон қолданбасын орнатқан кезде, барлық қажетті нәрсе ішінде жинақталған.

```
DiLink-Auto/
├── protocol/       Ортақ кітапхана (кадрлау, хабарламалар, анықтау, USB ADB)
├── app-client/     Телефон APK — реле, VD орналастыру, көлік авто-жаңарту, FileLog
├── app-server/     Көлік APK — UI, қосылу күй машинасы, бейне декодер
├── vd-server/      VirtualDisplay сервері (JAR-ға құрастырылған, телефонмен орналастырылады)
├── docs/           Құжаттама
└── gradle/         Құрастыру жүйесі
```

## Үлес Қосу

PR құпталады. Техникалық контекст үшін [Архитектура](./architecture.md) және [Протокол](./protocol.md) бөлімдерін қараңыз. `./gradlew :app-client:assembleDebug` арқылы құрастырыңыз (JDK 17+, Android SDK 34).

### Тармақтау Моделі (Git-Flow + Мәселе Түрлері)

Тармақтар пайдаланылған **мәселе үлгісіне** негізделіп мәселе агентімен автоматты түрде жасалады:

| Үлгі | Белгі | Тармақ Үлгісі | Мақсаты |
|----------|-------|---------------|---------|
| Қате Түзету | `bug` | `fix/N-agent` | Қателерді түзету |
| Жаңа Функция | `feature` | `feature/N-agent` | Жаңа функциялар |
| Зерттеу | `investigation` | `investigate/N-agent` | Код базасын зерттеу |
| Құжаттама | `documentation` | `docs/N-agent` | Құжаттама жаңартулары |
| Шығарылым | `release` | `release/vX.Y.Z` | Шығарылым дайындау |
| Агент Тапсырмасы (жалпы) | — | `issue/N-agent` | Жалпы |

`release/*` тармағынан басқа барлық тармақтар PR арқылы `develop` тармағына біріктіріледі, ол `main` тармағына бағытталады.

### CI Жұмыс Процестері

| Жұмыс Процесі | Триггер | Әрекет |
|----------|---------|--------|
| `build.yml` | `main` тармағына Push/PR | Тексеру: шығарылым APK құрастыру |
| `build-develop.yml` | `develop`, `release/*` тармағына Push/PR | Тексеру: жөндеу APK құрастыру |
| `build-pre-release.yml` | `vX.Y.Z-dev-NN` тегі | Жөндеу APK құрастыру + GitHub алдын ала шығарылым |
| `build-release.yml` | `vX.Y.Z` тегі | Қол қойылған шығарылым APK құрастыру + GitHub Release |
| `sync-main-to-develop.yml` | `main` тармағына Push | `main` → `develop` біріктіру (git-flow кері синхрондау) |
| `issue-agent.yml` | Мәселе ашылды / пікір | Автономды агент: тармақ, құрастыру, PR |

Барлық CI **өздігінен орналастырылған WSL раннерлерінде** жұмыс істейді.

**Шығарылым процесі:** Үлгіден Release мәселесін жасаңыз. Агент `release/vX.Y.Z` жасайды, өзгерістерді дайындайды және `vX.Y.Z-dev-NN` тегін қояды. Шығарылым тармағын жіберу `build-pre-release.yml` жұмыс процесін іске қосады, ол `git tag --points-at HEAD` арқылы коммиттегі `-dev` тегін тауып, алдын ала шығарылымды жариялайды. Дайын болғанда, `release/vX.Y.Z` біріктіру коммитінде `vX.Y.Z` тегімен `main` тармағына біріктіріледі. `main` тармағына Push `build-release.yml` (қол қойылған APK құрастырады + GitHub Release жасайды) және `sync-main-to-develop.yml` (`main` тармағын `develop` тармағына авто-біріктіреді) жұмыс процестерін іске қосады.

**Алдын ала шығарылым жаңартулары:** Алдын ала шығарылым арнасындағы пайдаланушылар `-dev` құрастырылымдарын алады. Шығарылым арнасындағы пайдаланушылар тек тұрақты құрастырылымдарды алады. Арна Параметрлерде бапталады.
