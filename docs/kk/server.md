# Көлік сервері қолданбасы (app-server)

## Шолу

Көлік сервері қолданбасы BYD DiLink ақпараттық ойын-сауық жүйесінде жұмыс істейді. Ол бір уақытта жұмыс істейтін WiFi (3 арнайы қосылым) және USB тректері бар **параллель қосылым моделін** пайдаланады:

1. Track A (WiFi): gateway IP + mDNS табу, control connect (9637), телефонмен handshake
2. Handshake-тен кейін: бейне (9638) + енгізу (9639) қосылымдары параллель ашылады
3. Track B (USB): құрылғыларды сканерлеу, USB ADB connect (logSink диагностикасымен), телефон қолданбасын іске қосу
4. `checkAndAdvance()` кез келген алғышарт өзгергенде күйді бағалайды
5. H.264 бейнені қабылдайды және көлік дисплейінде рендерлейді (экраннан тыс бетте декодерді ерте іске қосу)
6. Жанасу оқиғаларын алады және VD енгізуін енгізу үшін енгізу қосылымы арқылы телефонға жібереді

Күйлер: IDLE → CONNECTING → CONNECTED → STREAMING

Көлік APK телефон APK ішіне ендірілген. Телефон handshake кезінде нұсқа сәйкессіздігі анықталғанда көлік қолданбасын dadb арқылы автоматты түрде жаңартады. Көлік `UPDATING_CAR` хабарламасын алады және соқыр қайта қосылудың орнына күйді көрсетеді.

### Екі режимді UI

Көлік қолданбасы телефон қолданбасының тәсіліне сәйкес қосылу ағынын ағынды тәжірибеден екі бөлек режимге бөледі:

- **Launch mode** (`CarLaunchScreen`): Толық экранды, қосылуға бағытталған. Брендингті, қадамдық қосылу нұсқаулықтарын, қосылу күйін және қолмен IP енгізуді көрсетеді. Навигация панелі жоқ — барлық экран қосылуға арналған. Көлік қолданбасы іске қосылғанда көрсетіледі және телефоннан control қосылымы арқылы қолданба белгішелері келгенге дейін сақталады.

- **Streaming mode** (`CarShell` with `PersistentNavBar`): Сол жақ навигация панелі (хабарландырулар, басты бет, артқа, соңғы қолданбалар) және мазмұн аймағы (қолданбалар торы, айна көрінісі, хабарландырулар) бар таныс макет. `appList` бос емес және күй CONNECTED немесе STREAMING болғанда көрсетіледі.

Ауысу триггері: телефон control қосылымы арқылы `APP_LIST` жібергенде және қосылу күйі CONNECTED/STREAMING жеткенде, UI launch режимінен streaming режиміне ауысады.

## Компоненттер

### CarConnectionService

Параллель алғышарт күй машинасы және 3 арнайы қосылымы бар толық қосылу өмірлік циклін басқаратын алдыңғы план қызметі.

**3-Қосылым архитектурасы:**
- `controlConnection`: handshake, heartbeat, қолданба командалары, DATA арнасы (қолданбалар тізімі, хабарландырулар, көлік журналдары)
- `videoConnection`: тек H.264 бейне кадрлары (phone → car)
- `inputConnection`: тек жанасу оқиғалары (car → phone)
- Heartbeat/watchdog тек control қосылымында; бейне және енгізуде heartbeat үстеме шығыны жоқ
- Кез келген қосылымның үзілуі каскадты → толық сессияны тоқтату

**Параллель трек архитектурасы:**
- `connectionScope`: барлық табу coroutines үшін ата-аналық Job, ажыратылғанда бас тартылады
- Track A және Track B бір уақытта жұмыс істейді
- `checkAndAdvance()` кез келген алғышарт өзгергенде жалпы күйді бағалайды
- Қолданушы ажыратуы: IDLE күйінде қалады, авто-қайта қосылусыз (SharedPreferences-те сақталады)

**Track A — WiFi:**
- Табу: gateway IP (hotspot/LAN, әр 3s қайталанады) → mDNS → manual IP
- Control connection: телефон TCP:9637-ге NIO блокталмайтын SocketChannel connect
- Handshake: көрініс терезесі өлшемдері + DPI + appVersionCode + targetFps (60) жібереді → телефон ақпараты + vdServerJarPath алады
- Handshake жауабында: бейне (9638) + енгізу (9639) қосылымдарын параллель ашады, барлығы 3 орнатылғаннан кейін `wifiReady = true` орнатады
- Video: бейне қосылымы арқылы H.264 кадрларын алады, VideoDecoder-ге жібереді
- Touch: арнайы бір ағынды executor, `sendTouchEvent()` / `sendTouchBatch()` енгізу қосылымы арқылы
- Heartbeat: 3s интервал, 10s watchdog тайм-ауты (тек control қосылымы)
- Backoff: қайта қосылу сәтсіздіктерінде экспоненциалды кідіріс

**Track B — USB:**
- `USB_DEVICE_ATTACHED` / `USB_DEVICE_DETACHED` үшін `BroadcastReceiver` тіркейді
- `MainActivity` USB intent-терді қызметке жібереді
- USB құрылғыларын ADB интерфейсі үшін сканерлейді
- UsbAdbConnection арқылы USB ADB connect (protocol/ модулінде), барлық ADB auth журналдарын телефонның FileLog-ына бағыттайтын `logSink` арқылы
- Телефон қолданбасын іске қосады: `am start -n com.dilinkauto.client/.MainActivity`

**Жаңарту ағыны:**
- Телефон `UPDATING_CAR` жіберсе, көлік `updatingFromPhone = true` орнатады, "Updating car app..." күйін көрсетеді
- Жаңарту кезінде бейне/енгізу қосылымы әрекеттерін және қайта қосылу циклін өткізіп жібереді
- `pm install -r` кейін, көлік қолданбасы жаңадан іске қосылады

**Күй ағындары:**
- `_state`, `_phoneName`, `_appList`, `_notifications`, `_mediaMetadata`, `_playbackState`: UI үшін негізгі күй
- `_videoReady`: бірінші config емес бейне кадры келгенде true
- `_statusMessage`: UI көрсету үшін адам оқи алатын күй
- `_vdStackEmpty` (SharedFlow): телефон VD-де activity жоқ деп хабарлағанда шығарылады (басты бетке навигацияны іске қосады)

**VD Серверін іске қосу:**
- `deployVdServer()`: handshake-тің vdServerJarPath-пен CLASSPATH арқылы `shellNoWait`
- Args: `W H DPI PORT EW EH FPS` — FPS handshake-тің targetFps-нен беріледі

**Декодерді ерте іске қосу:**
- Бірінші CONFIG бейне кадрында, VideoDecoder-ді экраннан тыс SurfaceTexture-де іске қосады (MirrorScreen-ке дейін)
- MirrorScreen-нің `onSurfaceTextureAvailable` декодерді тоқтатып, нақты TextureView бетімен қайта іске қосады
- UI композиция кідірісі кезінде кілттік кадрлардың жоғалуын болдырмайды

**Көлік журналын бағыттау:**
- `carLogSend()` барлық көлік жағы журналдарын DATA арнасы `CAR_LOG` арқылы телефонға бағыттайды
- `videoDecoder.logSink` және `adb.setLogSink()` VideoDecoder және UsbAdbConnection журналдарын сол жолмен бағыттайды
- Барлық журналдар телефонның `/sdcard/DiLinkAuto/client.log` файлында көрінеді

### VideoDecoder

Surface шығысымен MediaCodec пайдаланатын H.264 декодері (GPU-тікелей рендерлеу).

- Кадр кезегі: 15 кадр — іске қосу жарысын және желі джиттерін буферлейді
- `onFrameReceived()`: `start()` шақырылғанға дейін де кадрларды кезекке қояды
- `start()`: алдымен кэштелген CONFIG-ті береді, содан кейін кезекті төгеді
- Кезек толғанда ең ескіні тастау: P-кадрларды тастауды қалайды, кілттік кадрлар/CONFIG үшін кезектегі P-кадрларды шығарады
- `KEY_LOW_LATENCY = 1`, `KEY_PRIORITY = 0` минималды декодтау кідірісі үшін
- CONFIG (SPS/PPS) кэштеледі және декодер қайта іске қосылғанда қайта ойнатылады
- Ерте іске қосуды үйлестіру үшін `isRunning` қасиеті
- `logSink` кері шақыруы барлық декодер журналдарын carLogSend арқылы телефонға бағыттайды
- Catchup режимі: кезек тереңдігіне негізделген төрт сатылы жылдамдату аймағы — normal (0-6 кадр), gentle 1.5x (7-12 кадр, кілттік емес 3-тің 1-ін өткізеді), medium 2x (13-20 кадр, 2-нің 1-ін өткізеді), aggressive 3x (21+ кадр, 3-тің 2-ін өткізеді). Кілттік кадрлар ешқашан өткізілмейді.
- 10+ ретті dequeueInputBuffer тастауларында кодекті тазартады және CONFIG-ті қайта береді

### ServerApp

Application класы. `IMPORTANCE_LOW` бар `dilinkauto_car_service` хабарландыру арнасын жасайды.

### RemoteAdbController

dadb кітапханасын пайдаланатын тікелей ADB клиенті. Виртуалды дисплейде shell командалары арқылы түрту, сырғыту, артқа, басты бетке және қолданбаны іске қосуды ұсынады. Балама енгізу жолы ретінде пайдаланылады.

### CarLaunchScreen

Телефон қосылымы орнатылғанға дейін көрсетілетін толық экранды қосылуға бағытталған composable — навигация панелі жоқ, қолданбалар торы жоқ.

- DiLink Auto брендингі (белгіше, тақырып, ұран)
- Түсті индикатор нүктесі (жасыл=streaming, қызғылт сары=connected/connecting, сұр=idle) және тікелей күй мәтіні бар қосылу күйі карточкасы
- "How to connect" нұсқаулықтары: 4 нөмірленген қадам (хотспотты қосу, USB қосу, телефон қолданбасын ашу, авто-қосылуды күту)
- Тікелей қосылу үшін қолмен IP енгізу
- `appList` бос емес болып, күй CONNECTED/STREAMING жеткенде ағынды режим макетімен ауыстырылады

### PersistentNavBar

76dp сол жақ навигация панелі — **тек ағынды режимде көрсетіледі** — мыналармен:
- Сағат дисплейі (HH:mm, әр 1s жаңартылады)
- Eject түймесі (ажыратады және қолданушы қалауын сақтайды)
- Желі күйінің индикаторы
- Оқылмаған саны бар хабарландырулар түймесі
- Home түймесі
- Back түймесі
- Соңғы қолданба белгішелері (ең көбі 5, қолданбалар қолжетімсіз болғанда тазаланады)
- 40dp белгішелер, 12-14sp мәтін

Ені H.264 кодтаушы үшін жұп көрініс терезесіне кепілдік беру үшін есептеледі.

### NotificationScreen

- Уақыт белгісі бойынша сұрыпталған хабарландырулар тізімі (ең жаңасы бірінші)
- ID бойынша дедубликация: жаңартулар барларын ауыстырады (прогресс хабарландыруларын өңдейді)
- Прогресс жолақтары: determinate (толтырылған) және indeterminate (айналмалы)
- Түрту арқылы іске қосу: хабарландыруды түрту VD-де ие қолданбаны іске қосады және айна көрінісіне ауысады

### Қолданбалар торы (HomeContent)

Ағынды режим белсенді және ағымдағы экран HOME болғанда негізгі мазмұн аймағы ретінде көрсетіледі:
- `imePadding()` бар іздеу өрісі — пернетақта activity-ді итермейді, тек іздеу жолағы қозғалады
- Манифестте `windowSoftInputMode="adjustNothing"`
- 160dp бейімделген тор ұяшықтарында 64dp қолданба белгішелері
- Қолданба атауы мәтіні: bodyLarge
- Әліпбилік сұрыптау
- Қолмен IP енгізу
- Қосылу күйі

### LauncherScreen (Ескі)

`CarStatusBar`, `SideNavBar` (80dp) және `AppGrid` бар толық интеграцияланған іске қосу макеті. Ағымдағы `CarShell` бағдарында пайдаланылмайды — белсенді UI `PersistentNavBar` + `HomeContent`/`MirrorContent`/`NotificationContent` composable-дарын пайдаланады.

### RecentAppsState

Соңғы іске қосылған қолданбаларды бақылайды (ең көбі 5), SharedPreferences-те сақталады. `pruneUnavailable()` қолданбалар тізімі жаңартылғанда бұдан былай жоқ қолданбаларды жояды.

### NavBarComponents

Жеке навигация панелі виджет composable-дары: `ClockDisplay` (әр 1s жаңартылады), `NetworkInfo` (connected/disconnected күйі), `RecentAppIcon` (40dp, белсенді күйді бөлектеумен), `NavActionButton` (40dp белгішелер, 12sp белгілер).

### CarTheme

Санат бойынша қолданба тақташа түстері бар Material3 күңгірт түс схемасы (`CarDark`): Navigation (жасыл), Music (қызғылт), Communication (көк), Other (сұр).

### Көлік дисплейі туралы ақпарат

BYD DiLink 3.0-де тексерілген:
- Экран: 1920x990 @ 240dpi
- Көрініс терезесі: ~1806x990 (76dp навигация панелінен кейін)
- VD: ~3282x1800 @ 480dpi (кодтау үшін ~1806x990 дейін GPU-масштабталған)

## Тәуелділіктер

- Jetpack Compose + Material 3
- Protocol module (телефон қолданбасымен ортақ, UsbAdbConnection + AdbProtocol + VideoConfig қамтиды)
- dadb 1.2.10 (TCP ADB резерві)
