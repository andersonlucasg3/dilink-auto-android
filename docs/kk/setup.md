# Баптау нұсқаулығы

## Алғышарттар

- **Телефон:** USB жөндеуі қосылған кез келген Android 10+ құрылғысы
- **Автомобиль:** BYD DiLink 3.0+ (немесе USB хост порты бар кез келген Android 10+ бас құрылғысы)
- **USB кабелі:** Телефонды автомобильдің USB портына қосу
- **Әзірлеу:** Android Studio немесе Gradle, JDK 17

## Телефонды баптау (бір реттік)

1. **DiLink Auto Client орнатыңыз**: `adb install app-client-debug.apk`
2. **Қолданбаны ашыңыз** — бастапқы баптау экраны сізді әрбір рұқсат арқылы бағыттайды:
   - **Барлық файлдарға қол жеткізу** — виртуалды дисплей серверін жадқа орналастырады
   - **Батареяны оңтайландыру** — экран өшірулі кезде ағындық таратуды белсенді күйде ұстайды
   - **Қолжетімділік қызметі** — автомобиль сенсорлық экранын басқаруды қосады
   - **Хабарландыруларға қол жеткізу** — телефон хабарландыруларын автомобиль дисплейіне жібереді
3. Әрбір қадам тиісті жүйелік параметрлерді ашады. Рұқсатты беріңіз, содан кейін жалғастыру үшін "Артқа" түймесін басыңыз.
4. Кез келген қадамды өткізіп жіберуге және оны кейінірек негізгі экраннан баптауға болады.

Болды. Сымсыз жөндеу, жұптау кодтары, арнайы WiFi баптауы қажет емес.

## Автомобильді баптау

Автомобильге қолмен орнату қажет емес. Автомобиль APK телефон APK ішіне ендірілген. Алғашқы қосылымда (немесе нұсқа сәйкессіздігінде) телефон автомобильге `UPDATING_CAR` жібереді (ол "Жаңарту..." күйін көрсетеді), содан кейін dadb (WiFi ADB) арқылы автоматты түрде орнатады.

Балама ретінде, автомобиль APK-сын қолмен жіберу үшін телефон қолданбасындағы "Автомобильге орнату" түймесін пайдаланыңыз.

Қолмен орнату кезінде: `adb install app-server-debug.apk` (автомобильге ADB қол жеткізуді қажет етеді).

## Күнделікті пайдалану

1. **Телефонды автомобильдің USB портына қосыңыз** — автомобиль қолданбасы автоматты түрде іске қосылады (немесе бір желіде WiFi арқылы қосылыңыз)
2. Автомобиль мен телефон параллель WiFi + USB тректері арқылы автоматты түрде қосылады
3. Телефон VD серверін орналастырады және ағындық тарату 60fps жылдамдықпен басталады
4. Телефон қолданбаларымен әрекеттесу үшін автомобильдің сенсорлық экранын пайдаланыңыз

## Құрастыру

```bash
# Телефон APK құрастыру (серверді құрастыруды + ендіруді автоматты түрде іске қосады)
./gradlew :app-client:assembleDebug

# APK орналасуы:
# app-client/build/outputs/apk/debug/app-client-debug.apk  (телефон -- ендірілген автомобиль APK-сын қамтиды)
```

Құрастыру жүйесі app-server-ді автоматты түрде құрастырады және оны app-client-ке ендіреді, сондықтан тек бір APK-ны қолмен орнату қажет.

## Бұл қалай жұмыс істейді

Телефон автомобильге қосылған кезде:

1. **Автомобиль параллель тректерді іске қосады** — WiFi табу және USB анықтау бір уақытта орындалады
2. **A трегі (WiFi):** шлюз IP + mDNS табу → телефонның басқару портына NIO қосылу (9637)
3. **B трегі (USB):** құрылғыларды сканерлеу → USB ADB қосылу → `am start` арқылы телефон қолданбасын іске қосу
4. **Қол алысу:** автомобиль viewport + DPI + appVersionCode + targetFps жібереді; телефон құрылғы ақпараты + vdServerJarPath жібереді
5. **Нұсқаны тексеру:** телефон appVersionCode салыстырады — сәйкессіздік болса, автомобильге UPDATING_CAR хабарламасын жібереді, содан кейін dadb арқылы автоматты түрде жаңартады
6. **3 қосылымды баптау:** автомобиль қол алысудан кейін бейне (9638) + енгізу (9639) қосылымдарын ашады
7. **Телефон VD серверін орналастырады** — vd-server.jar файлын `/sdcard/DiLinkAuto/` жолына шығарады, `app_process`-ті shell UID ретінде FPS аргументімен іске қосады
8. **VD сервері телефонға кері қосылады** localhost:19637 мекенжайы бойынша (NIO блокталмайтын)
9. **VD сервері VirtualDisplay жасайды** телефонның табиғи DPI (480dpi) мәнінде, GPU масштабтауды төмендетумен және мерзімді қайта салумен
10. **Бейне ағыны** WiFi TCP арқылы (бейне қосылымы 9638) — H.264, Негізгі профиль, 8Mbps CBR, 60fps дейін бапталады

## Ақаулықтарды жою

### ADB аутентификация диалогы әр уақытта пайда болады (v0.13.1 нұсқасында ТҮЗЕТІЛДІ)
Түзетілді — мәселе AUTH_TOKEN екі рет хэштеуден туындады. ADB шикі 20-байттық токенді жібереді, ол алдын ала хэштелген SHA-1 дайджесті ретінде қарастырылуы керек. Ескі код `SHA1withRSA` пайдаланды, ол оны қайта хэштеді. Енді SHA-1 DigestInfo префиксімен `NONEwithRSA` пайдаланылады (алдын ала хэштелген), бұл AOSP `RSA_sign(NID_sha1)` сәйкес келеді. Телефон қайта қосылу кезінде AUTH_SIGNATURE қабылдайды және "Әрқашан рұқсат ету" дұрыс сақталады.

Егер диалог жаңартудан кейін бірінші қосылымда әлі де пайда болса, "Әрқашан рұқсат ету" құсбелгісін қойыңыз — ол келесі қосылымдарда қайта пайда болмауы керек. Егер мәселе жалғаса берсе, телефонның Әзірлеуші параметрлерінде "ADB авторизация тайм-аутын өшіру" (Android 11+) тексеріңіз.

### Телефон қолданбасы іске қосылмайды
- Телефонда USB жөндеуі қосылғанына көз жеткізіңіз
- Автомобильдің USB порты хост режимін қолдайтынын тексеріңіз (барлық порттар қолдамайды)
- Басқа USB кабелін қолданып көріңіз (кейбір кабельдер тек зарядтауға арналған)

### Барлық файлдарға қол жеткізу рұқсаты берілмеді
- Телефон қолданбасына vd-server.jar файлын `/sdcard/DiLinkAuto/` жолына орналастыру үшін MANAGE_EXTERNAL_STORAGE қажет
- Параметрлер -> Қолданбалар -> DiLink Auto -> Рұқсаттар -> Барлық файлдарға қол жеткізу -> ҚОСУ тармағына өтіңіз

### Бейне ағыны жоқ / қара экран
- Телефон мен автомобильдің бір желіде екеніне көз жеткізіңіз
- Екі қолданбаның да жұмыс істеп тұрғанын тексеріңіз (телефон "Ағындық тарату" көрсетеді, автомобиль бейнені көрсетеді)
- VD серверіне іске қосылу үшін уақыт қажет болуы мүмкін — қосылғаннан кейін 5-10 секунд күтіңіз
- SurfaceScaler мерзімді қайта салуы статикалық мазмұнда да кадрларды шығаруы керек
- Диагностикалық ақпарат үшін `/sdcard/DiLinkAuto/client.log` тексеріңіз

### Қосылым үзіледі
- Бұрын байланыссыз желі үзілістері (мобильді деректердің ауысуы) белсенді ажыратуды тудырды
- v0.12.5 нұсқасында түзетілді: ақылды желі кері шақыруы қосылымды тасымалдамайтын желілердегі үзілістерді елемейді
- Егер мәселе жалғаса берсе, `/sdcard/DiLinkAuto/client.log` ішінде "Network lost" жазбаларын тексеріңіз

### Автомобиль қолданбасы жаңартылмайды
- Телефон автомобиль қолданбасын қол алысу кезінде нұсқа сәйкессіздігі анықталғанда автоматты түрде жаңартады
- Автомобиль жаңарту кезінде "Автомобиль қолданбасы жаңартылуда..." күйін көрсетеді
- Сонымен қатар, телефон қолданбасындағы "Автомобильге орнату" түймесі арқылы жаңартуды қолмен іске қосуға болады
- dadb автомобильге WiFi ADB арқылы қол жеткізе алатынына көз жеткізіңіз

### Журналдар
- Телефон журналдары: `/sdcard/DiLinkAuto/client.log` (ағымдағы сеанс)
- Алдыңғы сеанстар: `/sdcard/DiLinkAuto/client-YYYYMMDD-HHmmss.log`
- VD серверінің журналдары: `/data/local/tmp/vd-server.log` (телефонда, ADB арқылы оқуға болады)
- Автомобиль журналдары: протоколдың DATA арнасы арқылы телефонның client.log файлына бағытталады (тег: `CarLog`)
- Журналдарды алу: `adb shell "cat /sdcard/DiLinkAuto/client.log"`

## HyperOS (Xiaomi) кеңестері

HyperOS жүйесінде сенімді жұмыс істеу үшін:
1. Параметрлер -> Қолданбалар -> DiLink Auto -> Автоіске қосу -> Қосу
2. Параметрлер -> Батарея -> DiLink Auto -> Шектеусіз
3. Қолданбаны соңғы қолданбаларда бекітіңіз (картаны ұзақ басу -> Бекіту)
## Samsung One UI бойынша кеңестер

One UI 5+ (Android 13+) жүйесіндегі Samsung құрылғыларында DiLink-Auto дұрыс жұмыс істеуіне кедергі келтіретін қосымша қауіпсіздік және энергия үнемдеу мүмкіндіктері бар. Бұл Galaxy A, M, S, Z және Tab серияларына қолданылады.

### Auto Blocker өшіру (USB ADB үшін өте маңызды)

**Auto Blocker** USB командаларын бұғаттайды және автокөліктің телефонға USB ADB арқылы қосылуына кедергі келтіруі мүмкін. Бұл Samsung-қа тән ең кең таралған мәселе.

1. **Баптаулар → Қауіпсіздік және құпиялылық → Auto Blocker → Өшірулі**
2. Auto Blocker қосулы күйінде қалдыруды қаласаңыз, кем дегенде **"USB кабелі арқылы командаларды бұғаттау"** опциясын өшіріңіз

### Барлық файлдарға қол жеткізуге рұқсат беру

Samsung рұқсаттар менеджері сіз жақында ашпаған қолданбалардың рұқсаттарын автоматты түрде қайтарып алуы мүмкін:

1. **Баптаулар → Қолданбалар → DiLink Auto → Рұқсаттар → Файлдар және медиа → Барлық файлдарды басқаруға рұқсат беру**
2. **"Барлық файлдарды басқаруға рұқсат беру"** тумблерін қосыңыз
3. Баптауларды жапқаннан кейін оның ҚОСУЛЫ күйінде қалғанын тексеріңіз (Samsung растау поп-апын көрсетуі мүмкін)

### Батареяны оңтайландыруды өшіру

Samsung батареяны басқаруы стандартты Android-қа қарағанда агрессивті:

1. **Баптаулар → Қолданбалар → DiLink Auto → Батарея → Шектеусіз**
2. **Баптаулар → Батарея → Фондық пайдалану шектеулері → Ешқашан ұйықтамайтын қолданбалар → DiLink Auto қосу**
3. **Баптаулар → Батарея → Фондық пайдалану шектеулері → Терең ұйқыдағы қолданбалар → DiLink Auto жою** егер тізімде болса

### Қолданбаны соңғыларда бекіту

Samsung One UI жадыны босату үшін фондық қолданбаларды өшіруі мүмкін:

1. Соңғы қолданбаларды ашыңыз (3 түймелі навигациямен төменнен жоғары сырғытыңыз немесе жест навигациясы)
2. Картаның жоғарғы жағындағы DiLink Auto белгішесін түртіңіз
3. **"Ашық қалдыру"** тармағын таңдаңыз

### Samsung Device Care автоматты оңтайландыруын өшіру

Samsung Device Care фондық қызметтерді автоматты түрде тоқтатуы мүмкін:

1. **Баптаулар → Батарея және құрылғы күтімі → Автоматтандыру → Күнделікті автооңтайландыру → Өшірулі**
2. **Баптаулар → Батарея және құрылғы күтімі → Автоматтандыру → Автоқайта қосу → Өшірулі**

### "DeX" рұқсат поп-апын көрсеңіз

Кейбір Samsung құрылғылары қолданба виртуалды дисплей жасауға әрекеттенгенде "Samsung DeX" немесе "сыртқы дисплей" рұқсаттары туралы поп-ап көрсетеді. Galaxy A/M сериялары DeX қолдамаса да, диалог пайда болуы мүмкін. Жай ғана **"Рұқсат беру"** немесе **"Қазір бастау"** түймесін басыңыз. Егер диалог қайталана берсе, **Баптаулар → Қосылған құрылғылар → Samsung DeX** бөліміне өтіп, "HDMI қосылғанда автоматты іске қосу" опциясын өшіріңіз.

### Knox қауіпсіздік ескертулері

Samsung Knox келесілерге қол жеткізген кезде қауіпсіздік хабарландыруын көрсетуі мүмкін:
- Виртуалды дисплей беті (бейнені кодтау үшін)
- USB жөндеу көпірі (автокөлік ADB қосылымы үшін)
- Барлық файлдар қоймасы (VD серверін орналастыру үшін)

Бұл күтілетін әрекеттер. Knox-қа қатысты кез келген сұрауларда "Рұқсат беру" немесе "OK" түймесін басыңыз. Егер сұраулар тоқтамаса, Knox қорғанысын уақытша "Орташа" деңгейіне төмендетуге болады: **Баптаулар → Қауіпсіздік және құпиялылық → Samsung Knox**.
