# Посібник з налаштування

## Передумови

- **Телефон:** Будь-який пристрій Android 10+ з увімкненим налагодженням через USB
- **Автомобіль:** BYD DiLink 3.0+ (або будь-який головний пристрій Android 10+ з портом USB-хоста)
- **USB-кабель:** Від телефона до USB-порту автомобіля
- **Розробка:** Android Studio або Gradle, JDK 17

## Налаштування телефона (одноразово)

1. **Встановіть DiLink Auto Client**: `adb install app-client-debug.apk`
2. **Відкрийте застосунок** — екран початкового налаштування проведе вас через кожен дозвіл:
   - **Доступ до всіх файлів** — розгортає сервер віртуального дисплея у сховище
   - **Оптимізація батареї** — підтримує трансляцію при вимкненому екрані
   - **Спеціальні можливості** — вмикає сенсорне керування з автомобіля
   - **Доступ до сповіщень** — пересилає сповіщення телефона на дисплей автомобіля
3. Кожен крок відкриває відповідні системні налаштування. Надайте дозвіл, потім натисніть Назад, щоб продовжити.
4. Ви можете пропустити будь-який крок і налаштувати його пізніше на головному екрані.

Ось і все. Жодного налагодження через Wi-Fi, жодних кодів сполучення, жодного спеціального налаштування WiFi.

## Налаштування автомобіля

Ручне встановлення на автомобіль не потрібне. APK автомобіля вбудований в APK телефона. При першому підключенні (або незбігу версій) телефон надсилає `UPDATING_CAR` автомобілю (який показує статус "Оновлення..."), потім автоматично встановлює через dadb (WiFi ADB).

Як альтернативу використовуйте кнопку "Встановити в авто" у застосунку телефона для ручного надсилання APK автомобіля.

При ручному встановленні: `adb install app-server-debug.apk` (потребує доступу ADB до автомобіля).

## Щоденне використання

1. **Підключіть телефон до USB автомобіля** — застосунок автомобіля запускається автоматично (або підключіться через WiFi в одній мережі)
2. Автомобіль і телефон з'єднуються автоматично через паралельні треки WiFi + USB
3. Телефон розгортає VD-сервер, і трансляція починається з частотою 60 кадрів/с
4. Використовуйте сенсорний екран автомобіля для взаємодії із застосунками телефона

## Збірка

```bash
# Збірка APK телефона (автоматично запускає збірку сервера + вбудовування)
./gradlew :app-client:assembleDebug

# Розташування APK:
# app-client/build/outputs/apk/debug/app-client-debug.apk  (телефон -- включає вбудований APK автомобіля)
```

Система збірки автоматично збирає app-server і вбудовує його в app-client, тому вручну потрібно встановити лише один APK.

## Як це працює

Коли телефон підключено до автомобіля:

1. **Автомобіль запускає паралельні треки** — виявлення WiFi та USB виконуються одночасно
2. **Трек A (WiFi):** IP шлюзу + виявлення mDNS → NIO підключення до порту керування телефона (9637)
3. **Трек B (USB):** сканування пристроїв → підключення USB ADB → запуск застосунку телефона через `am start`
4. **Handshake:** автомобіль надсилає viewport + DPI + appVersionCode + targetFps; телефон надсилає інформацію про пристрій + vdServerJarPath
5. **Перевірка версії:** телефон порівнює appVersionCode — якщо не збігається, надсилає повідомлення UPDATING_CAR автомобілю, потім автоматично оновлює через dadb
6. **Налаштування 3 з'єднань:** автомобіль відкриває з'єднання video (9638) + input (9639) після handshake
7. **Телефон розгортає VD-сервер** — видобуває vd-server.jar у `/sdcard/DiLinkAuto/`, запускає `app_process` як shell UID з аргументом FPS
8. **VD-сервер робить зворотне підключення** до телефона на localhost:19637 (неблокувальний NIO)
9. **VD-сервер створює VirtualDisplay** з рідним DPI телефона (480dpi) з пониженням роздільної здатності GPU та періодичним перемальовуванням
10. **Відео передається** через WiFi TCP (відеоз'єднання 9638) — H.264, профіль Main, 8 Мбіт/с CBR, налаштовується до 60 кадрів/с

## Усунення несправностей

### Діалог автентифікації ADB з'являється щоразу (ВИПРАВЛЕНО у v0.13.1)
Виправлено — проблема була в подвійному хешуванні AUTH_TOKEN. ADB надсилає сирий 20-байтовий токен, який слід розглядати як попередньо хешований дайджест SHA-1. Старий код використовував `SHA1withRSA`, який хешував його знову. Тепер використовується `NONEwithRSA` з префіксом SHA-1 DigestInfo (попереднє хешування), що відповідає `RSA_sign(NID_sha1)` з AOSP. Телефон приймає AUTH_SIGNATURE при повторному підключенні, і "Завжди дозволяти" зберігається коректно.

Якщо діалог все ще з'являється при першому підключенні після оновлення, позначте "Завжди дозволяти" — він не повинен з'являтися знову. Якщо проблема зберігається, перевірте в налаштуваннях розробника опцію "Вимкнути тайм-аут авторизації ADB" (Android 11+).

### Застосунок телефона не запускається
- Переконайтеся, що налагодження через USB увімкнено на телефоні
- Перевірте, чи підтримує USB-порт автомобіля режим хоста (не всі порти підтримують)
- Спробуйте інший USB-кабель (деякі кабелі лише для зарядки)

### Дозвіл на доступ до всіх файлів відхилено
- Застосунку телефона потрібен MANAGE_EXTERNAL_STORAGE для розгортання vd-server.jar у `/sdcard/DiLinkAuto/`
- Перейдіть у Налаштування -> Застосунки -> DiLink Auto -> Дозволи -> Доступ до всіх файлів -> УВІМК

### Відео не транслюється / чорний екран
- Переконайтеся, що телефон і автомобіль знаходяться в одній мережі
- Перевірте, що обидва застосунки запущені (телефон показує "Трансляція", автомобіль показує відео)
- VD-серверу може знадобитися час для запуску — зачекайте 5-10 секунд після підключення
- SurfaceScaler з періодичним перемальовуванням повинен видавати кадри навіть на статичному вмісті
- Перевірте `/sdcard/DiLinkAuto/client.log` для діагностичної інформації

### Обриви з'єднання
- Раніше спричинялося незв'язаними обривами мережі (перемикання мобільних даних), що викликали активне відключення
- Виправлено у v0.12.5: розумний мережевий зворотний виклик ігнорує обриви на мережах, які не несуть з'єднання
- Якщо проблема зберігається, перевірте `/sdcard/DiLinkAuto/client.log` на наявність записів "Network lost"

### Застосунок автомобіля не оновлюється
- Телефон автоматично оновлює застосунок автомобіля при виявленні незбігу версій під час handshake
- Автомобіль показує статус "Оновлення застосунку авто..." під час оновлення
- Ви також можете вручну запустити оновлення кнопкою "Встановити в авто" у застосунку телефона
- Переконайтеся, що dadb може досягти автомобіля через WiFi ADB

### Логи
- Логи телефона: `/sdcard/DiLinkAuto/client.log` (поточна сесія)
- Попередні сесії: `/sdcard/DiLinkAuto/client-YYYYMMDD-HHmmss.log`
- Логи VD-сервера: `/data/local/tmp/vd-server.log` (на телефоні, читаються через ADB)
- Логи автомобіля: маршрутизуються в client.log телефона через канал DATA протоколу (тег: `CarLog`)
- Виймання логів: `adb shell "cat /sdcard/DiLinkAuto/client.log"`

## Поради для HyperOS (Xiaomi)

Для надійної роботи на HyperOS:
1. Налаштування -> Застосунки -> DiLink Auto -> Автозапуск -> Увімкнути
2. Налаштування -> Батарея -> DiLink Auto -> Без обмежень
3. Закріпіть застосунок у недавніх застосунках (довге натискання на картку -> Заблокувати)

## Поради для Samsung One UI

Пристрої Samsung з One UI 5+ (Android 13+) мають додаткові функції безпеки та енергозбереження, які можуть заважати правильній роботі DiLink-Auto. Це стосується серій Galaxy A, M, S, Z та Tab.

### Вимкнути Auto Blocker (критично важливо для USB ADB)

**Auto Blocker** блокує команди USB і може завадити автомобілю підключитися до телефона через USB ADB. Це найпоширеніша проблема, характерна для Samsung.

1. **Налаштування → Безпека та конфіденційність → Auto Blocker → Вимк**
2. Якщо ви віддаєте перевагу залишити Auto Blocker увімкненим, щонайменше вимкніть опцію **"Блокувати команди через USB-кабель"**

### Дозволити доступ до всіх файлів

Менеджер дозволів Samsung може автоматично відкликати дозволи в застосунків, які ви давно не відкривали:

1. **Налаштування → Застосунки → DiLink Auto → Дозволи → Файли та медіа → Дозволити керування всіма файлами**
2. Увімкніть тумблер **"Дозволити керування всіма файлами"**
3. Переконайтеся, що він залишається УВІМК після закриття налаштувань (Samsung може показати підтверджувальний поп-ап)

### Вимкнути оптимізацію батареї

Керування батареєю Samsung більш агресивне, ніж у стандартному Android:

1. **Налаштування → Застосунки → DiLink Auto → Батарея → Без обмежень**
2. **Налаштування → Батарея → Обмеження фонового використання → Ніколи не сплячі застосунки → Додати DiLink Auto**
3. **Налаштування → Батарея → Обмеження фонового використання → Застосунки в глибокому сні → Видалити DiLink Auto** якщо в списку

### Закріпити застосунок у недавніх

One UI Samsung може вбивати фонові застосунки для звільнення пам'яті:

1. Відкрийте Недавні застосунки (проведіть угору від нижньої частини з навігацією 3 кнопками або жестами)
2. Торкніться значка DiLink Auto у верхній частині картки
3. Виберіть **"Залишити відкритим"**

### Вимкнути автоматичну оптимізацію Samsung Device Care

Device Care Samsung може автоматично зупиняти фонові служби:

1. **Налаштування → Батарея та догляд за пристроєм → Автоматизація → Щоденна автооптимізація → Вимк**
2. **Налаштування → Батарея та догляд за пристроєм → Автоматизація → Автоперезавантаження → Вимк**

### Якщо ви бачите поп-ап дозволу "DeX"

Деякі пристрої Samsung показують поп-ап про "Samsung DeX" або дозволи "зовнішнього дисплея", коли застосунок намагається створити віртуальний дисплей. Хоча серії Galaxy A/M не підтримують DeX, діалог все одно може з'явитися. Просто натисніть **"Дозволити"** або **"Почати зараз"**. Якщо діалог продовжує з'являтися, перейдіть у **Налаштування → Підключені пристрої → Samsung DeX** і вимкніть "Автозапуск при підключенні HDMI".

### Зауваження щодо безпеки Knox

Samsung Knox може показувати сповіщення безпеки, коли DiLink-Auto отримує доступ до:
- Поверхні віртуального дисплея (для кодування відео)
- Моста налагодження USB (для підключення ADB автомобіля)
- Сховища всіх файлів (для розгортання VD-сервера)

Це очікувана поведінка. Натискайте "Дозволити" або "ОК" на будь-яких запитах, пов'язаних з Knox. Якщо запити не припиняються, можна тимчасово знизити захист Knox до "Середнього" в розділі **Налаштування → Безпека та конфіденційність → Samsung Knox**.
