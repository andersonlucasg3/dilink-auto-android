# Посібник з налаштування

## Передумови

- **Телефон:** Будь-який пристрій з Android 10+ із увімкненим налагодженням через USB
- **Автомобіль:** BYD DiLink 3.0+ (або будь-який головний пристрій з Android 10+ і USB-портом хоста)
- **USB-кабель:** Телефон до USB-порту автомобіля
- **Розробка:** Android Studio або Gradle, JDK 17

## Налаштування телефону (одноразове)

1. **Встановіть DiLink Auto Client**: `adb install app-client-debug.apk`
2. **Відкрийте додаток** — екран початкового налаштування проведе вас через кожен дозвіл:
   - **Доступ до всіх файлів** — розгортає сервер віртуального дисплея у сховище
   - **Оптимізація батареї** — підтримує потокову передачу, коли екран вимкнено
   - **Служба спеціальних можливостей** — вмикає керування сенсорним екраном автомобіля
   - **Доступ до сповіщень** — пересилає сповіщення телефону на дисплей автомобіля
3. Кожен крок відкриває відповідні системні налаштування. Надайте дозвіл, потім натисніть "Назад", щоб продовжити.
4. Ви можете пропустити будь-який крок і налаштувати його пізніше з головного екрана.

Це все. Жодного бездротового налагодження, жодних кодів сполучення, жодного спеціального налаштування WiFi.

## Налаштування автомобіля

Ручне встановлення на автомобіль не потрібне. APK для автомобіля вбудовано в APK телефону. При першому підключенні (або невідповідності версій) телефон надсилає `UPDATING_CAR` на автомобіль (який показує статус "Оновлення..."), потім автоматично встановлює через dadb (WiFi ADB).

Як альтернатива, використовуйте кнопку "Встановити на автомобіль" у додатку телефону, щоб вручну передати APK на автомобіль.

При ручному встановленні: `adb install app-server-debug.apk` (потребує доступу ADB до автомобіля).

## Щоденне використання

1. **Підключіть телефон до USB-порту автомобіля** — додаток автомобіля автоматично запускається (або підключіться через WiFi в тій самій мережі)
2. Автомобіль і телефон з'єднуються автоматично через паралельні треки WiFi + USB
3. Телефон розгортає сервер VD, і потокова передача починається зі швидкістю 60 кадрів на секунду
4. Використовуйте сенсорний екран автомобіля для взаємодії з додатками телефону

## Збірка

```bash
# Зібрати APK телефону (автоматично запускає збірку сервера + вбудовування)
./gradlew :app-client:assembleDebug

# Розташування APK:
# app-client/build/outputs/apk/debug/app-client-debug.apk  (телефон -- містить вбудований APK автомобіля)
```

Система збірки автоматично збирає app-server і вбудовує його в app-client, тому потрібно вручну встановити лише один APK.

## Як це працює

Коли телефон підключено до автомобіля:

1. **Автомобіль запускає паралельні треки** — виявлення WiFi і виявлення USB виконуються одночасно
2. **Трек A (WiFi):** IP шлюзу + виявлення mDNS → підключення NIO до порту керування телефону (9637)
3. **Трек B (USB):** сканування пристроїв → підключення USB ADB → запуск додатку телефону через `am start`
4. **Рукостискання:** автомобіль надсилає viewport + DPI + appVersionCode + targetFps; телефон надсилає інформацію про пристрій + vdServerJarPath
5. **Перевірка версії:** телефон порівнює appVersionCode — якщо невідповідність, надсилає повідомлення UPDATING_CAR на автомобіль, потім автоматично оновлює через dadb
6. **Налаштування 3 з'єднань:** автомобіль відкриває відео (9638) + введення (9639) з'єднання після рукостискання
7. **Телефон розгортає сервер VD** — видобуває vd-server.jar до `/sdcard/DiLinkAuto/`, запускає `app_process` як shell UID з аргументом FPS
8. **Сервер VD зворотно підключається** до телефону на localhost:19637 (NIO неблокуючий)
9. **Сервер VD створює VirtualDisplay** з нативним DPI телефону (480dpi) з апаратним зниженням роздільної здатності GPU і періодичним перемальовуванням
10. **Відео передається** через WiFi TCP (відеоз'єднання 9638) — H.264, основний профіль, 8 Мбіт/с CBR, налаштовується до 60 кадрів на секунду

## Усунення несправностей

### Діалог авторизації ADB з'являється щоразу (ВИПРАВЛЕНО у v0.13.1)
Виправлено — проблема полягала в подвійному хешуванні AUTH_TOKEN. ADB надсилає необроблений 20-байтовий токен, який слід розглядати як попередньо хешований дайджест SHA-1. Старий код використовував `SHA1withRSA`, який хешував його знову. Тепер використовується `NONEwithRSA` з префіксом DigestInfo SHA-1 (попередньо хешований), що відповідає `RSA_sign(NID_sha1)` з AOSP. Телефон приймає AUTH_SIGNATURE при повторному підключенні, і "Завжди дозволяти" зберігається коректно.

Якщо діалог все ще з'являється при першому підключенні після оновлення, встановіть прапорець "Завжди дозволяти" — він не повинен з'являтися знову при наступних підключеннях. Якщо проблема зберігається, перевірте в параметрах розробника телефону "Вимкнути тайм-аут авторизації ADB" (Android 11+).

### Додаток телефону не запускається
- Переконайтеся, що налагодження через USB увімкнено на телефоні
- Перевірте, чи підтримує USB-порт автомобіля режим хоста (не всі порти це роблять)
- Спробуйте інший USB-кабель (деякі кабелі лише для заряджання)

### Дозвіл "Доступ до всіх файлів" відхилено
- Додатку телефону потрібен MANAGE_EXTERNAL_STORAGE для розгортання vd-server.jar до `/sdcard/DiLinkAuto/`
- Перейдіть до Налаштування -> Додатки -> DiLink Auto -> Дозволи -> Доступ до всіх файлів -> УВІМК

### Відео не транслюється / чорний екран
- Переконайтеся, що телефон і автомобіль знаходяться в одній мережі
- Перевірте, що обидва додатки запущені (телефон показує "Потокова передача", автомобіль показує відео)
- Серверу VD може знадобитися час для запуску — зачекайте 5-10 секунд після підключення
- Періодичне перемальовування SurfaceScaler має створювати кадри навіть на статичному вмісті
- Перевірте `/sdcard/DiLinkAuto/client.log` для діагностичної інформації

### Втрата з'єднання
- Раніше спричинялося незв'язаними обривами мережі (циклічне перемикання мобільних даних), які запускали активне відключення
- Виправлено у v0.12.5: розумний зворотний виклик мережі ігнорує обриви на мережах, які не несуть з'єднання
- Якщо проблема зберігається, перевірте `/sdcard/DiLinkAuto/client.log` на наявність записів "Network lost"

### Додаток автомобіля не оновлюється
- Телефон автоматично оновлює додаток автомобіля при виявленні невідповідності версій під час рукостискання
- Автомобіль показує статус "Оновлення додатку автомобіля..." під час оновлення
- Ви також можете вручну запустити оновлення за допомогою кнопки "Встановити на автомобіль" у додатку телефону
- Переконайтеся, що dadb може досягти автомобіль через WiFi ADB

### Журнали
- Журнали телефону: `/sdcard/DiLinkAuto/client.log` (поточний сеанс)
- Попередні сеанси: `/sdcard/DiLinkAuto/client-YYYYMMDD-HHmmss.log`
- Журнали сервера VD: `/data/local/tmp/vd-server.log` (на телефоні, читаються через ADB)
- Журнали автомобіля: направляються до client.log телефону через канал DATA протоколу (тег: `CarLog`)
- Витягування журналів: `adb shell "cat /sdcard/DiLinkAuto/client.log"`

## Поради для HyperOS (Xiaomi)

Для надійної роботи на HyperOS:
1. Налаштування -> Додатки -> DiLink Auto -> Автозапуск -> Увімкнути
2. Налаштування -> Батарея -> DiLink Auto -> Без обмежень
3. Закріпіть додаток у нещодавніх додатках (довге натискання на картку -> Закріпити)
