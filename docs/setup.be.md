# Кіраўніцтва па наладцы

## Неабходныя ўмовы

- **Тэлефон:** Любая прылада Android 10+ з уключанай адладкай праз USB
- **Аўтамабіль:** BYD DiLink 3.0+ (або любая галаўная прылада Android 10+ з USB-хост портам)
- **USB-кабель:** Ад тэлефона да USB-порта аўтамабіля
- **Распрацоўка:** Android Studio або Gradle, JDK 17

## Наладка тэлефона (аднаразова)

1. **Усталюйце DiLink Auto Client**: `adb install app-client-debug.apk`
2. **Адкрыйце праграму** — экран пачатковай наладкі правядзе вас праз кожны дазвол:
   - **Доступ да ўсіх файлаў** — разгортвае сервер віртуальнага дысплея ў сховішча
   - **Аптымізацыя батарэі** — падтрымлівае трансляцыю пры выключаным экране
   - **Спецыяльныя магчымасці** — уключае сэнсарнае кіраванне з аўтамабіля
   - **Доступ да апавяшчэнняў** — перасылае апавяшчэнні тэлефона на дысплей аўтамабіля
3. Кожны крок адкрывае адпаведныя сістэмныя наладкі. Дайце дазвол, затым націсніце Назад, каб працягнуць.
4. Вы можаце прапусціць любы крок і наладзіць яго пазней на галоўным экране.

Вось і ўсё. Ніякай адладкі праз Wi-Fi, ніякіх кодаў спалучэння, ніякай спецыяльнай наладкі WiFi.

## Наладка аўтамабіля

Ручная ўстаноўка на аўтамабіль не патрэбна. APK аўтамабіля ўбудаваны ў APK тэлефона. Пры першым падключэнні (або несупадзенні версій) тэлефон адпраўляе `UPDATING_CAR` аўтамабілю (які паказвае статус "Абнаўленне..."), затым аўтаматычна ўсталёўвае праз dadb (WiFi ADB).

У якасці альтэрнатывы выкарыстоўвайце кнопку "Усталяваць у аўто" у праграме тэлефона для ручной адпраўкі APK аўтамабіля.

Пры ручной усталёўцы: `adb install app-server-debug.apk` (патрабуе доступу ADB да аўтамабіля).

## Штодзённае выкарыстанне

1. **Падключыце тэлефон да USB аўтамабіля** — праграма аўтамабіля запускаецца аўтаматычна (або падключыцеся праз WiFi у адной сетцы)
2. Аўтамабіль і тэлефон злучаюцца аўтаматычна праз паралельныя трэкі WiFi + USB
3. Тэлефон разгортвае VD-сервер, і трансляцыя пачынаецца з частатой 60 кадраў/с
4. Выкарыстоўвайце сэнсарны экран аўтамабіля для ўзаемадзеяння з праграмамі тэлефона

## Зборка

```bash
# Зборка APK тэлефона (аўтаматычна запускае зборку сервера + убудаванне)
./gradlew :app-client:assembleDebug

# Размяшчэнне APK:
# app-client/build/outputs/apk/debug/app-client-debug.apk  (тэлефон -- уключае ўбудаваны APK аўтамабіля)
```

Сістэма зборкі аўтаматычна збірае app-server і ўбудоўвае яго ў app-client, таму ўручную трэба ўсталяваць толькі адзін APK.

## Як гэта працуе

Калі тэлефон падключаны да аўтамабіля:

1. **Аўтамабіль запускае паралельныя трэкі** — выяўленне WiFi і USB выконваюцца адначасова
2. **Трэк A (WiFi):** IP шлюза + выяўленне mDNS → NIO падключэнне да порта кіравання тэлефона (9637)
3. **Трэк B (USB):** сканаванне прылад → падключэнне USB ADB → запуск праграмы тэлефона праз `am start`
4. **Handshake:** аўтамабіль адпраўляе viewport + DPI + appVersionCode + targetFps; тэлефон адпраўляе інфармацыю аб прыладзе + vdServerJarPath
5. **Праверка версіі:** тэлефон параўноўвае appVersionCode — калі не супадае, адпраўляе паведамленне UPDATING_CAR аўтамабілю, затым аўтаматычна абнаўляе праз dadb
6. **Наладка 3 злучэнняў:** аўтамабіль адкрывае злучэнні video (9638) + input (9639) пасля handshake
7. **Тэлефон разгортвае VD-сервер** — здабывае vd-server.jar у `/sdcard/DiLinkAuto/`, запускае `app_process` як shell UID з аргументам FPS
8. **VD-сервер робіць адваротнае падключэнне** да тэлефона на localhost:19637 (неблакавальны NIO)
9. **VD-сервер стварае VirtualDisplay** з родным DPI тэлефона (480dpi) з паніжэннем раздзяляльнасці GPU і перыядычнай перамалёўкай
10. **Відэа перадаецца** праз WiFi TCP (відэазлучэнне 9638) — H.264, профіль Main, 8 Мбіт/с CBR, наладжваецца да 60 кадраў/с

## Вырашэнне праблем

### Дыялог аўтэнтыфікацыі ADB з'яўляецца кожны раз (ВЫПРАЎЛЕНА ў v0.13.1)
Выпраўлена — праблема была ў двайным хэшаванні AUTH_TOKEN. ADB адпраўляе сыры 20-байтавы токен, які трэба разглядаць як папярэдне хэшаваны дайджэст SHA-1. Стары код выкарыстоўваў `SHA1withRSA`, які хэшаваў яго зноў. Цяпер выкарыстоўваецца `NONEwithRSA` з прэфіксам SHA-1 DigestInfo (папярэдняе хэшаванне), што адпавядае `RSA_sign(NID_sha1)` з AOSP. Тэлефон прымае AUTH_SIGNATURE пры паўторным падключэнні, і "Заўсёды дазваляць" захоўваецца карэктна.

Калі дыялог усё яшчэ з'яўляецца пры першым падключэнні пасля абнаўлення, адзначце "Заўсёды дазваляць" — ён не павінен з'яўляцца зноў. Калі праблема захоўваецца, праверце ў наладах распрацоўшчыка опцыю "Адключыць тайм-аўт аўтарызацыі ADB" (Android 11+).

### Праграма тэлефона не запускаецца
- Пераканайцеся, што адладка праз USB уключана на тэлефоне
- Праверце, ці падтрымлівае USB-порт аўтамабіля рэжым хоста (не ўсе парты падтрымліваюць)
- Паспрабуйце іншы USB-кабель (некаторыя кабелі толькі для зарадкі)

### Дазвол на доступ да ўсіх файлаў адхілены
- Праграме тэлефона патрэбны MANAGE_EXTERNAL_STORAGE для разгортвання vd-server.jar у `/sdcard/DiLinkAuto/`
- Перайдзіце ў Налады -> Праграмы -> DiLink Auto -> Дазволы -> Доступ да ўсіх файлаў -> УКЛ

### Відэа не транслюецца / чорны экран
- Пераканайцеся, што тэлефон і аўтамабіль знаходзяцца ў адной сетцы
- Праверце, што абедзве праграмы запушчаны (тэлефон паказвае "Трансляцыя", аўтамабіль паказвае відэа)
- VD-серверу можа спатрэбіцца час для запуску — пачакайце 5-10 секунд пасля падключэння
- SurfaceScaler з перыядычнай перамалёўкай павінен выдаваць кадры нават на статычным змесціве
- Праверце `/sdcard/DiLinkAuto/client.log` для дыягнастычнай інфармацыі

### Страты злучэння
- Раней выклікалася незвязанымі стратамі сеткі (пераключэнне мабільных даных), якія выклікаюць актыўнае адключэнне
- Выпраўлена ў v0.12.5: разумны сеткавы зваротны выклік ігнаруе страты на сетках, якія не нясуць злучэнне
- Калі праблема захоўваецца, праверце `/sdcard/DiLinkAuto/client.log` на наяўнасць запісаў "Network lost"

### Праграма аўтамабіля не абнаўляецца
- Тэлефон аўтаматычна абнаўляе праграму аўтамабіля пры выяўленні несупадзення версій падчас handshake
- Аўтамабіль паказвае статус "Абнаўленне праграмы аўто..." падчас абнаўлення
- Вы таксама можаце ўручную запусціць абнаўленне кнопкай "Усталяваць у аўто" у праграме тэлефона
- Пераканайцеся, што dadb можа дасягнуць аўтамабіля праз WiFi ADB

### Логі
- Логі тэлефона: `/sdcard/DiLinkAuto/client.log` (бягучая сесія)
- Папярэднія сесіі: `/sdcard/DiLinkAuto/client-YYYYMMDD-HHmmss.log`
- Логі VD-сервера: `/data/local/tmp/vd-server.log` (на тэлефоне, чытаюцца праз ADB)
- Логі аўтамабіля: маршрутызуюцца ў client.log тэлефона праз канал DATA пратакола (тэг: `CarLog`)
- Выманне логаў: `adb shell "cat /sdcard/DiLinkAuto/client.log"`

## Парады для HyperOS (Xiaomi)

Для надзейнай працы на HyperOS:
1. Налады -> Праграмы -> DiLink Auto -> Аўтазапуск -> Уключыць
2. Налады -> Батарэя -> DiLink Auto -> Без абмежаванняў
3. Замацуйце праграму ў нядаўніх праграмах (доўгі націск на картку -> Заблакіраваць)

## Парады для Samsung One UI

Прылады Samsung з One UI 5+ (Android 13+) маюць дадатковыя функцыі бяспекі і энергазберажэння, якія могуць перашкаджаць правільнай працы DiLink-Auto. Гэта адносіцца да серый Galaxy A, M, S, Z і Tab.

### Адключыць Auto Blocker (крытычна важна для USB ADB)

**Auto Blocker** блакуе каманды USB і можа перашкодзіць аўтамабілю падключыцца да тэлефона праз USB ADB. Гэта самая распаўсюджаная праблема, характэрная для Samsung.

1. **Налады → Бяспека і прыватнасць → Auto Blocker → Выкл**
2. Калі вы аддаеце перавагу пакінуць Auto Blocker уключаным, як мінімум адключыце опцыю **"Блакіраваць каманды праз USB-кабель"**

### Дазволіць доступ да ўсіх файлаў

Менеджар дазволаў Samsung можа аўтаматычна адклікаць дазволы ў праграм, якія вы даўно не адкрывалі:

1. **Налады → Праграмы → DiLink Auto → Дазволы → Файлы і медыя → Дазволіць кіраванне ўсімі файламі**
2. Уключыце тумблер **"Дазволіць кіраванне ўсімі файламі"**
3. Пераканайцеся, што ён застаецца УКЛ пасля закрыцця налад (Samsung можа паказаць пацвярджальны поп-ап)

### Адключыць аптымізацыю батарэі

Кіраванне батарэяй Samsung больш агрэсіўнае, чым у стандартным Android:

1. **Налады → Праграмы → DiLink Auto → Батарэя → Без абмежаванняў**
2. **Налады → Батарэя → Абмежаванні фонавага выкарыстання → Праграмы, якія ніколі не спяць → Дадаць DiLink Auto**
3. **Налады → Батарэя → Абмежаванні фонавага выкарыстання → Праграмы ў глыбокім сне → Выдаліць DiLink Auto** калі ў спісе

### Замацаваць праграму ў нядаўніх

One UI Samsung можа забіваць фонавыя праграмы для вызвалення памяці:

1. Адкрыйце Нядаўнія праграмы (правядзіце ўверх ад ніжняй часткі з навігацыяй 3 кнопкамі або жэстамі)
2. Націсніце на значок DiLink Auto у верхняй частцы карткі
3. Выберыце **"Пакінуць адкрытым"**

### Адключыць аўтаматычную аптымізацыю Samsung Device Care

Device Care Samsung можа аўтаматычна спыняць фонавыя службы:

1. **Налады → Батарэя і догляд за прыладай → Аўтаматызацыя → Штодзённая аўтааптымізацыя → Выкл**
2. **Налады → Батарэя і догляд за прыладай → Аўтаматызацыя → Аўтаперазагрузка → Выкл**

### Калі вы бачыце поп-ап дазволу "DeX"

Некаторыя прылады Samsung паказваюць поп-ап аб "Samsung DeX" або дазволах "вонкавага дысплея", калі праграма спрабуе стварыць віртуальны дысплей. Хаця серыі Galaxy A/M не падтрымліваюць DeX, дыялог усё роўна можа з'явіцца. Проста націсніце **"Дазволіць"** або **"Пачаць зараз"**. Калі дыялог працягвае з'яўляцца, перайдзіце ў **Налады → Падключаныя прылады → Samsung DeX** і адключыце "Аўтазапуск пры падключэнні HDMI".

### Заўвагі па бяспецы Knox

Samsung Knox можа паказваць апавяшчэнне бяспекі, калі DiLink-Auto атрымлівае доступ да:
- Паверхні віртуальнага дысплея (для кадзіравання відэа)
- Моста адладкі USB (для падключэння ADB аўтамабіля)
- Сховішча ўсіх файлаў (для разгортвання VD-сервера)

Гэта чаканыя паводзіны. Націскайце "Дазволіць" або "ОК" на любых запытах, звязаных з Knox. Калі запыты не спыняюцца, можна часова знізіць абарону Knox да "Сярэдняй" у раздзеле **Налады → Бяспека і прыватнасць → Samsung Knox**.
