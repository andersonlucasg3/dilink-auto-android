# Кіраўніцтва па наладцы

## Перадумовы

- **Тэлефон:** Любая прылада з Android 10+ з уключанай адладкай па USB
- **Аўтамабіль:** BYD DiLink 3.0+ (або любая галаўная прылада з Android 10+ і USB-портам хоста)
- **USB-кабель:** Тэлефон да USB-порта аўтамабіля
- **Распрацоўка:** Android Studio або Gradle, JDK 17

## Налада тэлефона (аднаразовая)

1. **Усталюйце DiLink Auto Client**: `adb install app-client-debug.apk`
2. **Адкрыйце праграму** — экран пачатковай наладкі правядзе вас праз кожны дазвол:
   - **Доступ да ўсіх файлаў** — разгортвае сервер віртуальнага дысплея ў сховішча
   - **Аптымізацыя батарэі** — падтрымлівае струменевую перадачу, калі экран выключаны
   - **Служба спецыяльных магчымасцей** — уключае кіраванне сэнсарным экранам аўтамабіля
   - **Доступ да апавяшчэнняў** — перасылае апавяшчэнні тэлефона на дысплей аўтамабіля
3. Кожны крок адкрывае адпаведныя сістэмныя налады. Дайце дазвол, затым націсніце "Назад", каб працягнуць.
4. Вы можаце прапусціць любы крок і наладзіць яго пазней з галоўнага экрана.

Гэта ўсё. Ніякай бесправадной адладкі, ніякіх кодаў спалучэння, ніякай спецыяльнай налады WiFi.

## Налада аўтамабіля

Ручная ўстаноўка на аўтамабіль не патрабуецца. APK для аўтамабіля ўбудаваны ў APK тэлефона. Пры першым падключэнні (або несупадзенні версій) тэлефон адпраўляе `UPDATING_CAR` на аўтамабіль (які паказвае статус "Абнаўленне..."), затым аўтаматычна ўсталёўвае праз dadb (WiFi ADB).

У якасці альтэрнатывы, выкарыстоўвайце кнопку "Усталяваць на аўтамабіль" у праграме тэлефона, каб уручную перадаць APK на аўтамабіль.

Пры ручной устаноўцы: `adb install app-server-debug.apk` (патрабуе доступ ADB да аўтамабіля).

## Штодзённае выкарыстанне

1. **Падключыце тэлефон да USB-порта аўтамабіля** — праграма аўтамабіля аўтаматычна запускаецца (або падключыцеся праз WiFi у той жа сетцы)
2. Аўтамабіль і тэлефон злучаюцца аўтаматычна праз паралельныя трэкі WiFi + USB
3. Тэлефон разгортвае сервер VD, і струменевая перадача пачынаецца з 60 кадраў у секунду
4. Выкарыстоўвайце сэнсарны экран аўтамабіля для ўзаемадзеяння з праграмамі тэлефона

## Зборка

```bash
# Сабраць APK тэлефона (аўтаматычна запускае зборку сервера + убудаванне)
./gradlew :app-client:assembleDebug

# Размяшчэнне APK:
# app-client/build/outputs/apk/debug/app-client-debug.apk  (тэлефон -- уключае ўбудаваны APK аўтамабіля)
```

Сістэма зборкі аўтаматычна збірае app-server і ўбудоўвае яго ў app-client, таму трэба ўручную ўсталяваць толькі адзін APK.

## Як гэта працуе

Калі тэлефон падключаны да аўтамабіля:

1. **Аўтамабіль запускае паралельныя трэкі** — выяўленне WiFi і выяўленне USB выконваюцца адначасова
2. **Трэк A (WiFi):** IP шлюза + выяўленне mDNS → падключэнне NIO да порта кіравання тэлефона (9637)
3. **Трэк B (USB):** сканаванне прылад → падключэнне USB ADB → запуск праграмы тэлефона праз `am start`
4. **Рукапацісканне:** аўтамабіль адпраўляе viewport + DPI + appVersionCode + targetFps; тэлефон адпраўляе інфармацыю аб прыладзе + vdServerJarPath
5. **Праверка версіі:** тэлефон параўноўвае appVersionCode — калі несупадзенне, адпраўляе паведамленне UPDATING_CAR на аўтамабіль, затым аўтаматычна абнаўляе праз dadb
6. **Наладка 3 злучэнняў:** аўтамабіль адкрывае відэа (9638) + уваход (9639) злучэнні пасля рукапаціскання
7. **Тэлефон разгортвае сервер VD** — здабывае vd-server.jar у `/sdcard/DiLinkAuto/`, запускае `app_process` як shell UID з аргументам FPS
8. **Сервер VD зваротна падключаецца** да тэлефона на localhost:19637 (NIO неблакавальны)
9. **Сервер VD стварае VirtualDisplay** з натыўным DPI тэлефона (480dpi) з апаратным паніжэннем разрознення GPU і перыядычнай перамалёўкай
10. **Відэа перадаецца** праз WiFi TCP (відэазлучэнне 9638) — H.264, галоўны профіль, 8 Мбіт/с CBR, наладжваецца да 60 кадраў у секунду

## Ліквідацыя няспраўнасцей

### Дыялог аўтарызацыі ADB з'яўляецца кожны раз (ВЫПРАЎЛЕНА ў v0.13.1)
Выпраўлена — праблема была ў двайным хэшаванні AUTH_TOKEN. ADB адпраўляе неапрацаваны 20-байтавы токен, які трэба разглядаць як папярэдне хэшаваны дайджэст SHA-1. Стары код выкарыстоўваў `SHA1withRSA`, які хэшаваў яго зноў. Цяпер выкарыстоўваецца `NONEwithRSA` з прэфіксам DigestInfo SHA-1 (папярэдне хэшаваны), што адпавядае `RSA_sign(NID_sha1)` з AOSP. Тэлефон прымае AUTH_SIGNATURE пры паўторным падключэнні, і "Заўсёды дазваляць" захоўваецца карэктна.

Калі дыялог усё яшчэ з'яўляецца пры першым падключэнні пасля абнаўлення, пастаўце галачку "Заўсёды дазваляць" — ён не павінен з'яўляцца зноў пры наступных падключэннях. Калі праблема захоўваецца, праверце ў параметрах распрацоўшчыка тэлефона "Адключыць тайм-аўт аўтарызацыі ADB" (Android 11+).

### Праграма тэлефона не запускаецца
- Пераканайцеся, што адладка па USB уключана на тэлефоне
- Праверце, ці падтрымлівае USB-порт аўтамабіля рэжым хоста (не ўсе парты гэта робяць)
- Паспрабуйце іншы USB-кабель (некаторыя кабелі толькі для зарадкі)

### Дазвол "Доступ да ўсіх файлаў" адхілены
- Праграме тэлефона патрэбны MANAGE_EXTERNAL_STORAGE для разгортвання vd-server.jar у `/sdcard/DiLinkAuto/`
- Перайдзіце ў Налады -> Праграмы -> DiLink Auto -> Дазволы -> Доступ да ўсіх файлаў -> УКЛ

### Відэа не струменіцца / чорны экран
- Пераканайцеся, што тэлефон і аўтамабіль знаходзяцца ў адной сетцы
- Праверце, што абедзве праграмы запушчаны (тэлефон паказвае "Струменевая перадача", аўтамабіль паказвае відэа)
- Серверу VD можа спатрэбіцца час для запуску — пачакайце 5-10 секунд пасля падключэння
- Перыядычная перамалёўка SurfaceScaler павінна ствараць кадры нават на статычным змесце
- Праверце `/sdcard/DiLinkAuto/client.log` для дыягнастычнай інфармацыі

### Страта злучэння
- Раней выклікалася несваяцкімі абрывамі сеткі (цыклічнае пераключэнне мабільных даных), якія запускалі актыўнае адключэнне
- Выпраўлена ў v0.12.5: разумны зваротны выклік сеткі ігнаруе абрывы на сетках, якія не нясуць злучэнне
- Калі праблема захоўваецца, праверце `/sdcard/DiLinkAuto/client.log` на наяўнасць запісаў "Network lost"

### Праграма аўтамабіля не абнаўляецца
- Тэлефон аўтаматычна абнаўляе праграму аўтамабіля пры выяўленні несупадзення версій падчас рукапаціскання
- Аўтамабіль паказвае статус "Абнаўленне праграмы аўтамабіля..." падчас абнаўлення
- Вы таксама можаце ўручную запусціць абнаўленне з дапамогай кнопкі "Усталяваць на аўтамабіль" у праграме тэлефона
- Пераканайцеся, што dadb можа дасягнуць аўтамабіль праз WiFi ADB

### Журналы
- Журналы тэлефона: `/sdcard/DiLinkAuto/client.log` (бягучы сеанс)
- Папярэднія сеансы: `/sdcard/DiLinkAuto/client-YYYYMMDD-HHmmss.log`
- Журналы сервера VD: `/data/local/tmp/vd-server.log` (на тэлефоне, чытаюцца праз ADB)
- Журналы аўтамабіля: накіроўваюцца ў client.log тэлефона праз канал DATA пратакола (тэг: `CarLog`)
- Выцягванне журналаў: `adb shell "cat /sdcard/DiLinkAuto/client.log"`

## Парады для HyperOS (Xiaomi)

Для надзейнай працы на HyperOS:
1. Налады -> Праграмы -> DiLink Auto -> Аўтазапуск -> Уключыць
2. Налады -> Батарэя -> DiLink Auto -> Без абмежаванняў
3. Замацуйце праграму ў апошніх праграмах (доўгі націск на картку -> Замацаваць)
