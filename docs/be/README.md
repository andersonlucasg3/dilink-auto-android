# DiLink-Auto

Выкарыстоўвайце праграмы свайго тэлефона на ўбудаваным экране аўтамабіля. З адкрытым зыходным кодам, не патрабуе сэрвісаў Google.

Альтэрнатыва Android Auto з адкрытым зыходным кодам для **любога тэлефона з Android 10+** ў пары з інфармацыйна-забаўляльнымі сістэмамі **BYD DiLink 3.0+**. Першапачаткова створана з-за адсутнасці падтрымкі Xiaomi HyperOS / кітайскай прашыўкі, але працуе ўніверсальна.

## Што Ён Робіць

DiLink-Auto перадае праграмы вашага тэлефона на дысплей аўтамабіля з поўным сэнсарным кіраваннем. Запускайце навігацыю, музыку, паведамленні — любыя праграмы на тэлефоне — непасрэдна з экрана аўтамабіля. Апавяшчэнні з'яўляюцца на навігацыйнай панэлі аўтамабіля з індыкатарамі прагрэсу. Відэа H.264 з частатой да 60 кадраў/с, 8 Мбіт/с CBR, экран тэлефона выключаны для эканоміі батарэі.

**Першапачатковая матывацыя:** ліквідацыя разрыву, калі ваш тэлефон не можа запускаць Android Auto (кітайская прашыўка, няма Google Play Services), а ваш аўтамабіль падтрымлівае толькі Android Auto (без CarWith, CarPlay або Carlife). Але DiLink-Auto працуе з любым Android-тэлефонам — з сэрвісамі Google ці без.

| Прылада | Праблема |
|--------|-------|
| Xiaomi 17 Pro Max (HyperOS 3, кітайская прашыўка) | Няма Android Auto — няма Google Play Services |
| BYD Destroyer 05 / King (рынак Бразіліі) | Толькі Android Auto на галаўной прыладзе |
| Любы тэлефон з Android 10+ | Працуе незалежна ад прашыўкі ці Play Services |

## Патрабаванні

**Тэлефон:**
- Любы тэлефон з Android 10+
- Уключана адладка праз USB (Параметры распрацоўшчыка)
- Дазвол на доступ да ўсіх файлаў (запытваецца пры першым запуску)

**Аўтамабіль:**
- BYD DiLink 3.0 або навей
- Адзін свабодны порт USB-A

**Кропка доступу тэлефона павінна быць уключана** — аўтамабіль падключаецца да WiFi-кропкі доступу вашага тэлефона. Без кодаў спалучэння, без уліковага запісу Google.

**Падключэнне да інтэрнэту не патрабуецца.** DiLink-Auto перадае ўсё лакальна праз кропку доступу WiFi вашага тэлефона — аўтамабіль і тэлефон маюць зносіны напрамую. Падключэнне да інтэрнэту неабходна толькі для праграм, якія працуюць на вашым тэлефоне (напрыклад, навігацыя, музыка), але не для самога DiLink-Auto.

## Як Гэта Працуе

1. **Уключыце кропку доступу** — Уключыце WiFi-кропку доступу на тэлефоне. Аўтамабіль падключаецца да яе.
2. **Падключыце** — Падключыце тэлефон да USB-порта аўтамабіля
3. **Аўтаматычная ўстаноўка** — Тэлефон усталёўвае праграму аўтамабіля праз WiFi ADB (толькі першы раз, адзін націск)
4. **Аўтаматычнае падключэнне** — 3 выдзеленыя WiFi TCP-патокі: відэа (порт 9638), сэнсарны ўвод (порт 9639) і кіраванне (порт 9637)
5. **Карыстайцеся праграмамі** — Запускайце любыя праграмы з лаўнчара аўтамабіля. Праграма працуе на тэлефоне, адлюстроўваецца на аўтамабілі і рэагуе на дотык

Тэлефон запускае вашы праграмы на віртуальным дысплеі, кадуе экран у відэа H.264 і перадае яго на аўтамабіль. Дотыкі на экране аўтамабіля адпраўляюцца назад на тэлефон і ўводзяцца як рэальныя сэнсарныя падзеі. Фізічны экран тэлефона застаецца выключаным (эканомія батарэі) і можа выкарыстоўвацца незалежна.

## Устаноўка

<a href="https://github.com/andersonlucasg3/dilink-auto-android/releases/latest"><img src="https://img.shields.io/github/v/release/andersonlucasg3/dilink-auto-android?label=Download%20Latest%20Release" alt="Download Latest Release"></a>

Спампуйце апошнюю версію або збярыце з зыходнага кода:

1. **Зборка:** `./gradlew :app-client:assembleDebug`
2. **Усталюйце** APK з `app-client/build/outputs/apk/debug/app-client-debug.apk` толькі на ваш тэлефон
3. **Уключыце адладку праз USB** на тэлефоне (Настройкі → Параметры распрацоўшчыка)
4. **Адкрыйце DiLink-Auto** на тэлефоне і дайце доступ да ўсіх файлаў па запыце
5. **Уключыце кропку доступу, затым падключыце да USB аўтамабіля** — праграма аўтамабіля аўтаматычна ўсталюецца пры першым запуску праз WiFi ADB

APK аўтамабіля і JAR VD-сервера ўбудаваны ў APK тэлефона — вам ніколі не трэба ўсталёўваць што-небудзь на аўтамабіль самастойна.

## Бягучы Статус

**Працуе:**
- Паток відэа H.264 60 кадраў/с (8 Мбіт/с CBR, профіль Main, наладжваецца праз рукапацісканне)
- Поўны сэнсарны ўвод (мультытач, сцісканне для маштабавання)
- Лаўнчар праграм з пошукам, алфавітнай сартаваннем, значкі 64dp, адаптыўная сетка
- Апавяшчэнні на экране аўтамабіля з індыкатарамі прагрэсу, націск для адкрыцця
- Самаабнаўленне праз GitHub Releases (рэліз) або перадрэлізы (адладка)
- **Падтрымка Shizuku**: падключэнне без ADB, бясшумнае самаабнаўленне праз pm install
- Аўтаабнаўленне: тэлефон выяўляе састарэлую праграму аўтамабіля (параўнанне назваў версій) і абнаўляе яе праз WiFi ADB
- Экран тэлефона выключаны падчас перадачы (эканомія батарэі)
- Кіраваная onboarding для ўсіх неабходных дазволаў
- Інтэрнацыяналізацыя: англійская, партугальская, руская, беларуская, французская, казахская, украінская, узбекская
- Аднаўленне дысплея пасля адключэння USB (v0.14.0+)
- Пратэставана на BYD DiLink 3.0 (1920x990) + Xiaomi 17 Pro Max (Android 16) + POCO F5

**Хутка з'явіцца:** аўдыёпаток, кіраванне мультымедыя, віджэты навігацыі

**Вядомыя абмежаванні:**
- Працэс VD-сервера перазапускаецца пры адключэнні USB (перападключаецца аўтаматычна).
- Кропку доступу трэба ўключаць уручную (абмежаванне Android 16).
- Часавыя візуальныя артэфакты — гонка перазапуску дэкодэра, аднаўляецца на наступным ключавым кадры (~1 с).
- Затрымка перадачы ~100-200 мс пад нагрузкай. CBR 8 Мбіт/с.
- Дысплей можа заставацца выключаным пасля рэзкага адключэння USB (выпраўлена ў v0.14.0).
- **Некаторыя праграмы не запаўняюць экран (letterbox/толькі партрэт).** DiLink-Auto люстрана адлюстроўвае віртуальны дысплей у альбомнай арыентацыі на экран аўтамабіля. Праграмы, якія не падтрымліваюць альбомную арыентацыю, будуць адлюстроўвацца з палосамі або вузкімі — гэта цалкам кантралюецца кожнай праграмай, а не DiLink-Auto. Нічога нельга зрабіць з боку люстравання.

## Структура Праекта

APK тэлефона (`app-client`) змяшчае як APK аўтамабіля (`app-server`), так і JAR VD-сервера. Калі вы ўстанаўліваеце праграму на тэлефон, усё неабходнае ўжо ўнутры.

```
DiLink-Auto/
├── protocol/       Агульная бібліятэка (кадрыраванне, паведамленні, выяўленне, USB ADB)
├── app-client/     APK тэлефона — рэтранслятар, разгортванне VD, аўтаабнаўленне аўтамабіля, FileLog
├── app-server/     APK аўтамабіля — UI, стан машыны злучэння, відэадэкодэр
├── vd-server/      Сервер VirtualDisplay (скампіляваны ў JAR, разгортваецца тэлефонам)
├── docs/           Дакументацыя
└── gradle/         Сістэма зборкі
```

### Мадэль Галінавання (Git-Flow + Тыпы Задач)

Галіны ствараюцца аўтаматычна агентам задач на аснове выкарыстанага **шаблону задачы**:

| Шаблон | Пазнака | Узор Галіны | Прызначэнне |
|----------|-------|---------------|---------|
| Выпраўленне Памылкі | `bug` | `fix/N-agent` | Выпраўленне памылак |
| Новая Функцыя | `feature` | `feature/N-agent` | Новыя функцыі |
| Даследаванне | `investigation` | `investigate/N-agent` | Даследаванне кодавай базы |
| Дакументацыя | `documentation` | `docs/N-agent` | Абнаўленне дакументацыі |
| Рэліз | `release` | `release/vX.Y.Z` | Падрыхтоўка рэлізу |
| Задача Агента (агульная) | — | `issue/N-agent` | Агульнае |

Усе галіны зліваюцца ў `develop` праз PR, акрамя `release/*`, якая накіроўваецца на `main`.

### CI Працэсы

| Працэс | Трыгер | Дзеянне |
|----------|---------|--------|
| `build.yml` | Push/PR у `main` | Праверка: зборка рэлізнага APK |
| `build-develop.yml` | Push/PR у `develop`, `release/*` | Праверка: зборка адладачнага APK |
| `build-pre-release.yml` | Тэг `vX.Y.Z-dev-NN` | Зборка адладачнага APK + GitHub перадрэліз |
| `build-release.yml` | Тэг `vX.Y.Z` | Зборка падпісанага рэлізнага APK + GitHub Release |
| `sync-main-to-develop.yml` | Push у `main` | Зліццё `main` → `develop` (зваротная сінхранізацыя git-flow) |
| `issue-agent.yml` | Задача адкрыта / каментар | Аўтаномны агент: галіна, зборка, PR |

Увесь CI працуе на **самастойна размешчаных WSL раннерах**.

**Працэс рэлізу:** Стварыце задачу Release з шаблону. Агент стварае `release/vX.Y.Z`, рыхтуе змены і пазначае тэгам `vX.Y.Z-dev-NN`. Адпраўка галіны рэлізу запускае `build-pre-release.yml`, які знаходзіць тэг `-dev` на каміце праз `git tag --points-at HEAD` і публікуе перадрэліз. Калі гатовы, `release/vX.Y.Z` зліваецца ў `main` з тэгам `vX.Y.Z` на каміце зліцця. Push у `main` запускае `build-release.yml` (зборка падпісанага APK + стварэнне GitHub Release) і `sync-main-to-develop.yml` (аўтаматычнае зліццё `main` назад у `develop`).

**Абнаўленні перадрэлізаў:** Карыстальнікі на канале перадрэлізаў атрымліваюць зборкі `-dev`. Карыстальнікі на канале рэлізаў атрымліваюць толькі стабільныя зборкі. Канал наладжваецца ў Настройках.
