# Архитектура

## Обзор

DiLink-Auto — это проект Gradle из четырёх модулей:

```
DiLink-Auto/
├── protocol/        Android библиотека -- общая для обоих приложений (модуль Gradle)
├── app-client/      Android приложение -- работает на телефоне (модуль Gradle)
├── app-server/      Android приложение -- работает в автомобиле (модуль Gradle)
├── vd-server/       Android библиотека -- сервер VirtualDisplay (модуль Gradle)
```

## Архитектура Virtual Display

**Телефон** развёртывает и запускает сервер VD локально. Сервер VD подключается обратно к приложению телефона (обратное соединение на localhost:19637, полностью неблокирующий NIO). Приложения отображаются на VirtualDisplay с родным DPI телефона (480dpi), с GPU-масштабированием до разрешения viewport автомобиля для кодирования H.264. Физический экран телефона полностью независим.

## Почему USB ADB со стороны автомобиля?

`app_process` должен работать как shell UID (2000) для создания VirtualDisplays, способных размещать сторонние приложения. Автомобиль подключается к `adbd` телефона через режим USB хоста, используя собственную реализацию протокола ADB (`UsbAdbConnection` в модуле protocol/).

**Текущий подход:** Автомобиль выступает как USB ADB хост. Телефону нужно только **USB Debugging** (стандартная опция разработчика). Никакого Wireless Debugging, кодов сопряжения, зависимости от WiFi для настройки.

## Обязанности модулей

### protocol (Android библиотека)

Общая для обоих приложений. Содержит UsbAdbConnection, AdbProtocol, VideoConfig и NioReader. Ноль зависимостей помимо Kotlin корутин.

| Компонент | Файл | Назначение |
|-----------|------|---------|
| Кодек кадров | `FrameCodec.kt` | Бинарное кодирование/декодирование кадров, переиспользуемый буфер заголовка, NIO writeAll |
| Каналы | `Channel.kt` | ID каналов (control, video, audio, data, input) |
| Типы сообщений | `MessageType.kt` | Байтовые константы, включая `VD_STACK_EMPTY`, `UPDATING_CAR` |
| Сообщения | `Messages.kt` | Сериализуемые data class (handshake включает `appVersionCode`, `vdServerJarPath`, `targetFps`) |
| Соединение | `Connection.kt` | TCP соединение с опциональным heartbeat/watchdog, очередь записи без блокировок, NioReader |
| VideoConfig | `VideoConfig.kt` | `TARGET_FPS`, `FRAME_INTERVAL_MS` — общие константы синхронизации |
| NioReader | `NioReader.kt` | Неблокирующий читатель на основе Selector, настраиваемый таймаут select |
| Обнаружение | `Discovery.kt` | Регистрация/обнаружение сервиса mDNS, константы портов (9637/9638/9639) |
| UsbAdbConnection | `adb/UsbAdbConnection.java` | Протокол ADB через USB (CNXN, AUTH, OPEN, WRTE), обратный вызов logSink |
| AdbProtocol | `adb/AdbProtocol.java` | Константы сообщений ADB и сериализация |

### app-client (Приложение телефона)

Управляет развёртыванием сервера VD, автообновлением автомобиля, ретрансляцией по 3 соединениям и FileLog.

| Компонент | Файл | Назначение |
|-----------|------|---------|
| ConnectionService | `service/ConnectionService.kt` | Приём 3 портов (9637/9638/9639), развёртывание JAR VD, автообновление авто, умный сетевой обратный вызов |
| VirtualDisplayClient | `display/VirtualDisplayClient.kt` | NIO приём на localhost:19637, ретрансляция видео (videoConnection), пересылка касаний, пустой стек (controlConnection) |
| NotificationService | `service/NotificationService.kt` | Захват и пересылка уведомлений телефона с прогрессом |
| InputInjectionService | `service/InputInjectionService.kt` | Резервный вариант инжекции касаний (физический экран) |
| FileLog | `FileLog.kt` | Файловое логирование в `/sdcard/DiLinkAuto/client.log`, ротация, обход фильтрации logcat |
| MainActivity | `MainActivity.kt` | UI — старт/стоп, статус разрешений, кнопка Установить на автомобиль |

### app-server (Приложение автомобиля)

Параллельная модель соединения с WiFi (3 соединения) и USB дорожками.

| Компонент | Файл | Назначение |
|-----------|------|---------|
| AppIconCache | `AppIconCache.kt` | Кэш иконок на стороне автомобиля — декодирует исходные PNG 192x192 один раз, `prepareAll()` изменяет размер всех иконок в фоновом потоке, `getPrepared()` это O(1) поиск в ConcurrentHashMap без I/O при прокрутке |
| CarConnectionService | `service/CarConnectionService.kt` | Параллельный конечный автомат, 3 WiFi соединения + USB дорожка, обработка UPDATING_CAR |
| VideoDecoder | `decoder/VideoDecoder.kt` | Декодирование H.264, очередь 30 кадров, ранний старт на внеэкранной поверхности, обратный вызов logSink |
| CarLaunchScreen | `ui/screen/CarLaunchScreen.kt` | Полноэкранный экран запуска/подключения (без навигации), брендинг, инструкции, ручной IP |
| MirrorScreen | `ui/screen/MirrorScreen.kt` | TextureView + пересылка касаний, перезапуск декодера при доступности поверхности |
| HomeContent | `ui/screen/HomeScreen.kt` | Сетка приложений (значки 64dp, ячейки 160dp) или статус соединения, в режиме стриминга |
| LauncherScreen | `ui/screen/LauncherScreen.kt` | Устаревший интегрированный экран с SideNavBar, CarStatusBar, AppGrid |
| NotificationScreen | `ui/screen/NotificationScreen.kt` | Список уведомлений с индикаторами прогресса, запуск по нажатию |
| PersistentNavBar | `ui/nav/PersistentNavBar.kt` | Панель навигации 76dp (значки 40dp, текст 14sp), недавние приложения (с очисткой), только в режиме стриминга |
| RecentAppsState | `ui/nav/RecentAppsState.kt` | Отслеживает недавние приложения, удаляет недоступные |
| MainActivity | `MainActivity.kt` | Полноэкранный иммерсивный режим, пересылка USB intent, двухрежимная маршрутизация экранов (запуск vs стриминг) |

### vd-server (Привилегированный процесс shell)

Модуль библиотеки Android (`com.android.library`), компилируется через `bundleLibRuntimeToJarDebug`, затем D8 в JAR задачей `buildVdServer` в `app-client/build.gradle.kts`. Зависит от `:protocol` и `kotlinx-coroutines-core`. Развёртывается телефоном в `/sdcard/DiLinkAuto/`.

| Компонент | Файл | Назначение |
|-----------|------|---------|
| VirtualDisplayServer | `VirtualDisplayServer.kt` | Создаёт VD, очередь записи NIO + читатель Selector, кодер H.264, настраиваемый FPS, обратное давление |
| FakeContext | `FakeContext.kt` | Подделывает `com.android.shell` для доступа к DisplayManager |
| SurfaceScaler | `SurfaceScaler.kt` | Конвейер GPU-уменьшения EGL/GLES, пропускает GL работу при бездействии (полагается на repeat-previous-frame-after кодера) |

## Поток соединения

```
1. Телефон и автомобиль в одной сети (или телефон подключён к авто по USB)
2. Приложение авто запускается, начинает параллельные WiFi + USB дорожки

   Дорожка A (WiFi):
   a. Обнаружение IP шлюза + поиск mDNS
   b. NIO подключение к порту управления телефона (9637)
   c. Handshake: авто отправляет viewport + DPI + appVersionCode + targetFps
   d. Телефон отвечает информацией об устройстве + vdServerJarPath
   e. Телефон проверяет appVersionCode -- если разница, отправляет UPDATING_CAR, автообновляет через dadb
   f. Авто подключает видео (9638) + ввод (9639) параллельно после handshake
   g. Телефон принимает оба, сессия полностью установлена

   Дорожка B (USB):
   a. Сканирование USB устройств на интерфейс ADB
   b. USB ADB подключение (CNXN -> AUTH -> подключено), logSink направляет в carLogSend
   c. Запуск приложения телефона через am start

3. Телефон развёртывает vd-server.jar в /sdcard/DiLinkAuto/
4. Авто запускает: CLASSPATH=jar app_process / VirtualDisplayServer W H DPI PORT EW EH FPS
5. Сервер VD создаёт VirtualDisplay (DPI телефона) + GPU SurfaceScaler (периодическая перерисовка)
6. Сервер VD обратно подключается К телефону на localhost:19637 (неблокирующий NIO)
7. Телефон принимает соединение сервера VD через NIO ServerSocketChannel
8. Авто запускает VideoDecoder на внеэкранной поверхности сразу при первом CONFIG кадре
9. MirrorScreen отображается, перезапускает декодер с реальной поверхностью TextureView
10. Видеопоток: VD -> SurfaceScaler -> кодер -> очередь записи NIO -> localhost -> NioReader телефона -> очередь записи videoConnection -> WiFi TCP -> NioReader авто -> VideoDecoder -> TextureView
```

Состояния: IDLE -> CONNECTING -> CONNECTED -> STREAMING

## Ключевые проектные решения

| Решение | Обоснование |
|----------|-----------|
| **3 выделенных TCP соединения** | Управление/видео/ввод на отдельных сокетах. Устраняет межканальное обратное давление (список приложений не может заблокировать видео). |
| **Параллельные WiFi + USB дорожки** | Обе работают одновременно; checkAndAdvance() оценивает состояние при изменении любого предусловия. |
| **Телефон развёртывает JAR VD** | Не требуется push со стороны авто. Телефон извлекает vd-server.jar на sdcard через deployAssets(). |
| **Обратное VD соединение** | Сервер VD подключается К телефону (а не телефон к VD), упрощая обработку firewall/NAT. |
| **NIO неблокирующий везде** | Все сокеты неблокирующие: WiFi соединения, сервер VD localhost, чтение на основе Selector. Никакого блокирующего I/O в конвейере. |
| **Настраиваемый FPS** | Авто отправляет `targetFps` в handshake, сервер VD использует его. Все таймауты конвейера основаны на `FRAME_INTERVAL_MS = 1000/fps`. |
| **Encoder repeat-previous-frame-after** | SurfaceScaler пропускает GL работу на холостых кадрах. Кодер повторяет последний кадр до 500мс на статичном контенте, предотвращая голодание без накладных расходов GPU. |
| **Ранний старт декодера** | Декодер запускается на внеэкранном SurfaceTexture при первом CONFIG, до создания TextureView у MirrorScreen. Предотвращает потерю ключевых кадров при компоновке UI. |
| **Умный сетевой обратный вызов** | `onLost` игнорирует несвязанные потери сети (мобильные данные). Разрывает сессию только при потере сети активного соединения. |
| **Автообновление авто с сообщениями** | Телефон отправляет UPDATING_CAR перед установкой. Авто показывает статус, не переподключается слепо. |
| **APK авто встроен в APK телефона** | Система сборки упаковывает app-server.apk внутрь app-client, обеспечивая функцию Установить на автомобиль. |
| **GPU SurfaceScaler** | VD рендерит с 480dpi телефона (без масштабирования совместимости). GPU уменьшает до viewport авто. |
| **FakeContext** | ActivityThread + getSystemContext() для реального системного Context. Обходит NPE UserManager через рефлексию mDisplayIdToMirror. |
| **Флаги доверенного VD** | `OWN_DISPLAY_GROUP` + `OWN_FOCUS` + `TRUSTED` предотвращают миграцию активности. |
| **Чётная ширина viewport** | Ширина панели навигации подобрана для обеспечения чётных размеров, совместимых с H.264. |
| **Heartbeat только на управлении** | Видео и ввод не имеют накладных расходов heartbeat. Watchdog управляющего соединения обнаруживает мёртвых участников. |
| **FileLog** | Обходит фильтрацию logcat HyperOS. Файловое логирование с ротацией в `/sdcard/DiLinkAuto/`. |
| **Кэш иконок приложений** | Кэш на стороне автомобиля сохраняет исходные PNG (192x192) на диск. `prepareAll()` декодирует и изменяет размер всех иконок до размера сетки в фоновом потоке после получения APP_LIST. `getPrepared()` это поиск ConcurrentHashMap с нулевой стоимостью — без корутин, без I/O, без декодирования при прокрутке. |
| **Отклонение/очистка уведомлений** | Кнопка отклонения для каждого элемента и заголовок "Очистить все" на экране уведомлений автомобиля. Автомобиль отправляет сообщения данных NOTIFICATION_CLEAR / NOTIFICATION_CLEAR_ALL на телефон, который очищает соответствующие уведомления Android. |
| **Обратные вызовы logSink** | VideoDecoder и UsbAdbConnection направляют логи через протокол в FileLog телефона. |
| **Предварительно хешированная аутентификация ADB** | AUTH_SIGNATURE использует `NONEwithRSA` + префикс SHA-1 DigestInfo (предварительно хешировано). Соответствует `RSA_sign(NID_sha1)` из AOSP. "Всегда разрешать" сохраняется корректно. |
| **Питание экрана через SurfaceControl** | `DisplayControl` загружается из `services.jar` через `ClassLoaderFactory` (Android 14+). Резервный вариант `cmd display power-off/on`. Телефон восстанавливает экран при отключении VD. |
| **Catchup декодера** | Четыре зоны градуированного ускорения: нормальная (0-6 кадров), мягкая 1.5x (7-12), средняя 2x (13-20), агрессивная 3x (21+). Ключевые кадры никогда не пропускаются. |
| **Дедупликация запуска приложений** | `am start` без `--activity-clear-task`. Существующие приложения возобновляются вместо перезапуска. |
| **Обратное давление сервера VD** | Отбрасывает неключевые кадры на кодере при превышении очередью записи 6 кадров. Предотвращает неограниченный рост памяти. |
| **Отключение пользователем** | Остаётся в IDLE, без автопереподключения. Сохраняется в SharedPreferences. |

## Технологический стек

| Уровень | Технология |
|-------|-----------|
| Язык | Kotlin 1.9.22 (все модули) |
| Сборка | Gradle 8.7, AGP 8.2.2 |
| UI | Jetpack Compose + Material 3 |
| Видео | MediaCodec H.264 (кодер: сервер VD 8Mbps CBR Main, декодер: авто) |
| GPU | EGL14 + GLES20 + SurfaceTexture (SurfaceScaler с периодической перерисовкой) |
| Сеть | NIO ServerSocketChannel / SocketChannel / Selector, Android NSD (mDNS) |
| USB ADB | Собственный протокол в модуле protocol/ (общий), logSink для диагностики |
| WiFi ADB | dadb 1.2.10 (автообновление авто) |
| Async | Kotlin Coroutines + Flow |
| Мин. API | 29 (Android 10) |
| Версия приложения | versionName сравнивается через semver (общий в gradle.properties); versionCode всё ещё отправляется для обратной совместимости |
| Версия протокола | PROTOCOL_VERSION = 1 |
