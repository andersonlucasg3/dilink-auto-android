# Руководство по настройке

## Предварительные требования

- **Телефон:** Любое устройство Android 10+ с включенной отладкой по USB
- **Автомобиль:** BYD DiLink 3.0+ (или любое головное устройство Android 10+ с USB-портом хоста)
- **USB-кабель:** Кабель от телефона к USB-порту автомобиля
- **Разработка:** Android Studio или Gradle, JDK 17

**Подключение к интернету не требуется** — DiLink-Auto передаёт всё локально через точку доступа WiFi вашего телефона (автомобиль и телефон общаются напрямую). Подключение к интернету необходимо только для приложений, работающих на вашем телефоне (например, карты, музыка), но не для самого DiLink-Auto.

## Настройка телефона (Однократная)

1. **Установите DiLink Auto Client**: `adb install app-client-debug.apk`
2. **Откройте приложение** — экран первоначальной настройки проведет вас через каждое разрешение:
   - **Доступ ко всем файлам** — развертывает сервер виртуального дисплея в хранилище
   - **Оптимизация батареи** — поддерживает потоковую передачу при выключенном экране
   - **Служба специальных возможностей** — включает управление сенсорным экраном автомобиля
   - **Доступ к уведомлениям** — пересылает уведомления телефона на дисплей автомобиля
3. Каждый шаг открывает соответствующие системные настройки. Предоставьте разрешение, затем нажмите Назад для продолжения.
4. Вы можете пропустить любой шаг и настроить его позже с главного экрана.

Вот и всё. Никакой отладки по WiFi, никаких кодов сопряжения, никакой специальной настройки WiFi.

### Дополнительно: Shizuku (подключение без ADB)

[Shizuku](https://github.com/RikkaApps/Shizuku) предоставляет обычным приложениям повышенные привилегии оболочки, позволяя DiLink-Auto развертывать сервер виртуального дисплея без USB ADB от автомобиля. Это альтернатива USB ADB — автомобилю требуется только WiFi.

1. **Установите Shizuku** из [GitHub Releases](https://github.com/RikkaApps/Shizuku/releases) или Google Play
2. **Запустите Shizuku** один раз через ADB (`adb shell sh /sdcard/Android/data/moe.shizuku.privileged.api/start.sh`) или через root
3. **Откройте DiLink Auto** → Настройки → Shizuku → нажмите для предоставления разрешения
4. Когда Shizuku активен, телефон развертывает VD-сервер напрямую. USB ADB или отладка по WiFi не требуются.

При активном Shizuku автомобилю требуется только WiFi — USB-кабель для потоковой передачи не нужен.

## Настройка автомобиля

**Подключение к интернету не требуется.** APK-файл автомобиля встроен в APK-файл телефона — телефон отправляет его на автомобиль через локальный WiFi. Подключение к интернету не требуется ни на каком этапе настройки или потоковой передачи.

Ручная установка на автомобиль не требуется. APK-файл автомобиля встроен в APK-файл телефона. При первом подключении (или несовпадении версий) телефон отправляет `UPDATING_CAR` автомобилю (который показывает статус «Обновление...»), а затем автоматически устанавливает через dadb (WiFi ADB).

В качестве альтернативы используйте кнопку «Установить на автомобиль» в приложении телефона для ручной отправки APK-файла автомобиля.

При ручной установке: `adb install app-server-debug.apk` (требуется доступ ADB к автомобилю).

## Ежедневное использование

1. **Подключите телефон к USB-порту автомобиля** — приложение автомобиля запускается автоматически (или подключитесь через WiFi в той же сети)
2. Автомобиль и телефон соединяются автоматически через параллельные каналы WiFi + USB
3. Телефон развертывает VD-сервер, и потоковая передача начинается с частотой 60 кадров/с
4. Используйте сенсорный экран автомобиля для взаимодействия с приложениями телефона

## Сборка

```bash
# Сборка APK телефона (автоматически запускает сборку сервера + встраивание)
./gradlew :app-client:assembleDebug

# Расположение APK:
# app-client/build/outputs/apk/debug/app-client-debug.apk  (телефон -- включает встроенный APK автомобиля)
```

Система сборки автоматически компилирует app-server и встраивает его в app-client, поэтому вручную нужно установить только один APK-файл.

## Как это работает

Когда телефон подключен к автомобилю:

1. **Автомобиль запускает параллельные каналы** — обнаружение WiFi и обнаружение USB выполняются одновременно
2. **Канал A (WiFi):** IP шлюза + обнаружение mDNS → подключение NIO к порту управления телефона (9637)
3. **Канал B (USB):** сканирование устройств → подключение USB ADB → запуск приложения телефона через `am start`
4. **Рукопожатие:** автомобиль отправляет viewport + DPI + appVersionCode + targetFps; телефон отправляет информацию об устройстве + vdServerJarPath
5. **Проверка версии:** телефон сравнивает appVersionCode — при несовпадении отправляет сообщение UPDATING_CAR автомобилю, затем автоматически обновляет через dadb
6. **Настройка 3 соединений:** автомобиль открывает соединения видео (9638) + ввода (9639) после рукопожатия
7. **Телефон развертывает VD-сервер** — извлекает vd-server.jar в `/sdcard/DiLinkAuto/`, запускает `app_process` как shell UID с аргументом FPS
8. **VD-сервер выполняет обратное подключение** к телефону на localhost:19637 (неблокирующий NIO)
9. **VD-сервер создает VirtualDisplay** с нативным DPI телефона (480dpi) с масштабированием на GPU и периодической перерисовкой
10. **Потоковая передача видео** через WiFi TCP (видеосоединение 9638) — H.264, профиль Main, 8 Мбит/с CBR, настраиваемая до 60 кадров/с

## Устранение неполадок

### Диалог аутентификации ADB появляется каждый раз (ИСПРАВЛЕНО в v0.13.1)
Исправлено — проблема заключалась в двойном хешировании AUTH_TOKEN. ADB отправляет необработанный 20-байтовый токен, который должен рассматриваться как предварительно вычисленный дайджест SHA-1. Старый код использовал `SHA1withRSA`, который хешировал его снова. Теперь используется `NONEwithRSA` с префиксом SHA-1 DigestInfo (предварительно вычисленным), что соответствует `RSA_sign(NID_sha1)` из AOSP. Телефон принимает AUTH_SIGNATURE при повторном подключении, и «Всегда разрешать» сохраняется корректно.

Если диалог все еще появляется при первом подключении после обновления, отметьте «Всегда разрешать» — он не должен появляться снова при последующих подключениях. Если проблема сохраняется, проверьте в Параметрах разработчика телефона опцию «Отключить тайм-аут авторизации ADB» (Android 11+).

### Приложение телефона не запускается
- Убедитесь, что отладка по USB включена на телефоне
- Проверьте, поддерживает ли USB-порт автомобиля режим хоста (не все порты поддерживают)
- Попробуйте другой USB-кабель (некоторые кабели предназначены только для зарядки)

### Разрешение на доступ ко всем файлам отклонено
- Приложению телефона требуется MANAGE_EXTERNAL_STORAGE для развертывания vd-server.jar в `/sdcard/DiLinkAuto/`
- Перейдите в Настройки -> Приложения -> DiLink Auto -> Разрешения -> Доступ ко всем файлам -> ВКЛ

### Видео не транслируется / черный экран
- Убедитесь, что телефон и автомобиль находятся в одной сети
- Убедитесь, что оба приложения запущены (телефон показывает «Трансляция», автомобиль показывает видео)
- VD-серверу может потребоваться время для запуска — подождите 5-10 секунд после подключения
- Периодическая перерисовка SurfaceScaler должна создавать кадры даже на статичном содержимом
- Проверьте `/sdcard/DiLinkAuto/client.log` для диагностической информации

### Обрывы соединения
- Ранее вызывалось несвязанными обрывами сети (переключением мобильных данных), вызывающими упреждающее отключение
- Исправлено в v0.12.5: интеллектуальный сетевой обратный вызов игнорирует обрывы в сетях, не передающих соединение
- Если проблема сохраняется, проверьте `/sdcard/DiLinkAuto/client.log` на наличие записей «Network lost»

### Приложение автомобиля не обновляется
- Телефон автоматически обновляет приложение автомобиля при обнаружении несовпадения версий во время рукопожатия
- Автомобиль показывает статус «Обновление приложения автомобиля...» во время обновления
- Вы также можете вручную запустить обновление с помощью кнопки «Установить на автомобиль» в приложении телефона
- Убедитесь, что dadb может подключиться к автомобилю через WiFi ADB

### Журналы
- Журналы телефона: `/sdcard/DiLinkAuto/client.log` (текущий сеанс)
- Предыдущие сеансы: `/sdcard/DiLinkAuto/client-YYYYMMDD-HHmmss.log`
- Журналы VD-сервера: `/data/local/tmp/vd-server.log` (на телефоне, читаются через ADB)
- Журналы автомобиля: направляются в client.log телефона через канал DATA протокола (тег: `CarLog`)
- Извлечение журналов: `adb shell "cat /sdcard/DiLinkAuto/client.log"`

## Советы по HyperOS (Xiaomi)

Для надежной работы на HyperOS:
1. Настройки -> Приложения -> DiLink Auto -> Автозапуск -> Включить
2. Настройки -> Батарея -> DiLink Auto -> Без ограничений
3. Закрепите приложение в Недавних приложениях (долгое нажатие на карточку -> Закрепить)
## Советы для Samsung One UI

Устройства Samsung с One UI 5+ (Android 13+) имеют дополнительные функции безопасности и энергосбережения, которые могут препятствовать правильной работе DiLink-Auto. Это относится к сериям Galaxy A, M, S, Z и Tab.

### Отключить Auto Blocker (критически важно для USB ADB)

**Auto Blocker** блокирует команды USB и может помешать автомобилю подключиться к телефону через USB ADB. Это самая распространенная проблема, характерная для Samsung.

1. **Настройки → Безопасность и конфиденциальность → Auto Blocker → Выкл**
2. Если вы предпочитаете оставить Auto Blocker включенным, как минимум отключите опцию **"Блокировать команды по USB-кабелю"**

### Разрешить доступ ко всем файлам

Менеджер разрешений Samsung может автоматически отзывать разрешения у приложений, которые вы давно не открывали:

1. **Настройки → Приложения → DiLink Auto → Разрешения → Файлы и медиа → Разрешить управление всеми файлами**
2. Включите тумблер **"Разрешить управление всеми файлами"**
3. Убедитесь, что он остаётся ВКЛ после закрытия настроек (Samsung может показать подтверждающий поп-ап)

### Отключить оптимизацию батареи

Управление батареей Samsung более агрессивно, чем в стандартном Android:

1. **Настройки → Приложения → DiLink Auto → Батарея → Без ограничений**
2. **Настройки → Батарея → Ограничения фонового использования → Никогда не спящие приложения → Добавить DiLink Auto**
3. **Настройки → Батарея → Ограничения фонового использования → Приложения в глубоком сне → Удалить DiLink Auto** если в списке

### Закрепить приложение в недавних

One UI Samsung может убивать фоновые приложения для освобождения памяти:

1. Откройте Недавние приложения (проведите вверх от нижней части с навигацией 3 кнопками или жестами)
2. Нажмите на иконку DiLink Auto в верхней части карточки
3. Выберите **"Оставить открытым"**

### Отключить автоматическую оптимизацию Samsung Device Care

Device Care Samsung может автоматически останавливать фоновые службы:

1. **Настройки → Батарея и уход за устройством → Автоматизация → Ежедневная автооптимизация → Выкл**
2. **Настройки → Батарея и уход за устройством → Автоматизация → Автоперезагрузка → Выкл**

### Если вы видите поп-ап разрешения "DeX"

Некоторые устройства Samsung показывают поп-ап о "Samsung DeX" или разрешениях "внешнего дисплея", когда приложение пытается создать виртуальный дисплей. Хотя серии Galaxy A/M не поддерживают DeX, диалог всё равно может появиться. Просто нажмите **"Разрешить"** или **"Начать сейчас"**. Если диалог продолжает появляться, перейдите в **Настройки → Подключенные устройства → Samsung DeX** и отключите "Автозапуск при подключении HDMI".

### Замечания по безопасности Knox

Samsung Knox может показывать уведомление безопасности, когда DiLink-Auto получает доступ к:
- Поверхности виртуального дисплея (для кодирования видео)
- Мосту отладки USB (для подключения ADB автомобиля)
- Хранилищу всех файлов (для развертывания VD-сервера)

Это ожидаемое поведение. Нажимайте "Разрешить" или "ОК" на любых запросах, связанных с Knox. Если запросы не прекращаются, можно временно снизить защиту Knox до "Средней" в разделе **Настройки → Безопасность и конфиденциальность → Samsung Knox**.
