# Спецификация протокола

## Обзор

DiLink-Auto использует собственный бинарный протокол через **3 выделенных TCP соединения** между телефоном и автомобилем:

| Соединение | Порт | Направление | Содержимое |
|------------|------|-----------|---------|
| **Управление** | 9637 | Двунаправленное | Handshake, heartbeat, команды приложений, DATA (список приложений, уведомления, логи авто, медиа) |
| **Видео** | 9638 | Телефон → Авто | Только H.264 CONFIG + FRAME |
| **Ввод** | 9639 | Авто → Телефон | Только события касаний |

Каждое соединение имеет собственный сокет, NioReader и очередь записи — полная изоляция I/O предотвращает задержки видео из-за невидео трафика (например, больших пакетов списка приложений).

Управляющее соединение устанавливается первым (здесь происходит handshake). После handshake автомобиль открывает видео и ввод соединения параллельно. Телефон принимает все три и связывает их как одну сессию. Heartbeat/watchdog работает только на управляющем соединении; видео и ввод не имеют накладных расходов heartbeat. Отказ любого соединения вызывает каскадный разрыв всей сессии.

Отдельный внутренний протокол работает между приложением телефона и сервером VD на `localhost:19637` (документирован в [client.md](./client.md)). Сервер VD обратно подключается к телефону (телефон слушает, сервер VD подключается). Сервер VD использует полностью неблокирующий NIO для чтения и записи на сокете localhost.

## Формат данных

```
+---------------+------------+--------------+-----------------+
| Длина кадра    | ID канала  | Тип сообщения| Полезная нагрузка|
| (4 байта)      | (1 байт)   | (1 байт)     | (N байт)         |
| big-endian     |            |              |                  |
+---------------+------------+--------------+-----------------+
```

- **Длина кадра**: `uint32` big-endian. Значение = `2 + размер_полезной_нагрузки`.
- **Макс. нагрузка**: 128 МБ.
- **Накладные расходы заголовка**: 6 байт на кадр.

## Каналы

| ID | Имя | Соединение | Направление | Назначение |
|----|------|------------|-----------|---------|
| 0 | CONTROL | Управление (9637) | Двунаправленное | Handshake, heartbeat, команды приложений, сигналы сервера VD |
| 1 | VIDEO | Видео (9638) | Телефон → Авто | Кодированные видеокадры H.264 (ретранслируются от сервера VD) |
| 2 | AUDIO | (зарезервировано) | Телефон → Авто | Зарезервировано (не реализовано) |
| 3 | DATA | Управление (9637) | Двунаправленное | Уведомления, список приложений, метаданные медиа, логи авто |
| 4 | INPUT | Ввод (9639) | Авто → Телефон | События касаний (CMD_INPUT_TOUCH для сырой инжекции MotionEvent) |

## Канал управления (0x00)

### HANDSHAKE_REQUEST (0x01) -- Авто -> Телефон

```
+----------------------+------+
| protocolVersion       | int32 |
| длина deviceName       | int16 |
| deviceName            | UTF-8 |
| screenWidth           | int32 |  ширина дисплея авто в пикселях (после панели навигации)
| screenHeight          | int32 |  высота дисплея авто в пикселях
| supportedFeatures     | int32 |  битовая маска
| displayMode           | byte  |  0=MIRROR, 1=VIRTUAL (по умолчанию)
| screenDpi             | int32 |  плотность дисплея авто (напр. 240)
| appVersionCode        | int32 |  код версии приложения авто (устаревший, для обратной совместимости)
| targetFps             | int32 |  запрашиваемый авто FPS (напр. 60)
| длина appVersionName   | int16 |  длина строки versionName
| appVersionName        | UTF-8 |  имя версии приложения авто (напр. "0.16.0")
+----------------------+------+
```

### HANDSHAKE_RESPONSE (0x02) -- Телефон -> Авто

```
+----------------------+------+
| protocolVersion       | int32 |
| accepted              | byte  |  1=принято, 0=отклонено
| длина deviceName       | int16 |
| deviceName            | UTF-8 |
| displayWidth          | int32 |  ширина VD (соответствует запросу авто)
| displayHeight         | int32 |  высота VD (соответствует запросу авто)
| virtualDisplayId      | int32 |  -1 (устанавливается сервером VD, не телефоном)
| adbPort               | int32 |  5555 (TCP порт ADB телефона)
| vdServerJarPath       | UTF-8 |  путь к развёрнутому JAR VD на телефоне (напр. /sdcard/DiLinkAuto/vd-server.jar)
+----------------------+------+
```

### HEARTBEAT (0x03) / HEARTBEAT_ACK (0x04) -- Только управляющее соединение

Пустая нагрузка. Отправляется каждые 3 секунды по управляющему соединению. Если кадр не получен в течение 10 секунд, соединение считается мёртвым (таймаут watchdog). Видео и ввод соединения не имеют heartbeat.

### DISCONNECT (0x05) -- Двунаправленное

Пустая нагрузка. Корректное завершение.

### APP_STOPPED (0x14) -- Телефон -> Авто

Пустая нагрузка. Отправляется при остановке приложения на виртуальном дисплее.

### VD_SERVER_READY (0x20) -- Авто -> Телефон

Пустая нагрузка. Авто подтверждает, что процесс сервера VD работает на телефоне.

### LAUNCH_APP (0x10) -- Авто -> Телефон

```
+----------------------+------+
| packageName           | UTF-8 |  сырые байты, без префикса длины
+----------------------+------+
```

Телефон пересылает серверу VD, который выполняет `am start --display <id> -n <component>` (без `--activity-clear-task` — существующие приложения возобновляются).

### GO_HOME (0x11) / GO_BACK (0x12) -- Авто -> Телефон

Пустая нагрузка. Телефон пересылает серверу VD. GO_BACK отправляет `input -d <id> keyevent 4`, затем проверяет пустой стек. GO_HOME — пустая операция (авто обрабатывает навигацию запуска).

### APP_STARTED (0x13) -- Телефон -> Авто

Тот же формат, что и LAUNCH_APP. Подтверждает запуск приложения.

### VD_STACK_EMPTY (0x15) -- Телефон -> Авто

Пустая нагрузка. Отправляется после GO_BACK, когда сервер VD обнаруживает отсутствие оставшихся задач приложений на виртуальном дисплее (через `dumpsys activity activities`). Авто использует это для переключения с зеркального вида на домашний экран.

### FOCUSED_APP (0x16) -- Телефон -> Авто

Нагрузка: UTF-8 package name. Отправляется, когда приложение получает фокус на виртуальном дисплее. Автомобиль использует это для обновления состояния отслеживания приложений.

### APP_INFO (0x17) -- Авто -> Телефон

Нагрузка: UTF-8 package name. Автомобиль запрашивает телефон открыть системный экран информации/настроек для указанного пакета.

### APP_SHORTCUTS (0x18) -- Авто -> Телефон

Нагрузка: UTF-8 package name. Автомобиль запрашивает доступные ярлыки приложений Android 7.1+ для указанного пакета. **Отключено в UI** — инфраструктура (запрос VD-сервера + APK XML fallback) на месте, но ярлыки скрыты до доработки (issue #57).

### APP_SHORTCUTS_LIST (0x19) -- Телефон -> Авто

Нагрузка: `AppShortcutsListMessage` — package name + список дескрипторов ярлыков (id, shortLabel, longLabel). Отправляется в ответ на запрос APP_SHORTCUTS.

### APP_SHORTCUT_ACTION (0x1A) -- Авто -> Телефон

Нагрузка: `AppShortcutActionMessage` — package name + shortcut id. Запускает конкретный ярлык на виртуальном дисплее.

### APP_UNINSTALL (0x1B) -- Авто -> Телефон

Нагрузка: UTF-8 package name. Автомобиль запрашивает телефон удалить указанный пакет. Телефон обрабатывает системный диалог удаления и отправляет обратно `APP_UNINSTALLED` через канал данных после завершения.

### UPDATING_CAR (0x30) -- Телефон -> Авто

Пустая нагрузка. Отправляется перед тем, как телефон начинает автообновление приложения авто. Автомобиль показывает "Обновление приложения автомобиля..." и останавливает переподключение. После обновления приложение авто перезапускается заново.

## Видео канал (0x01)

### CONFIG (0x01) -- Телефон -> Авто

NAL единицы SPS/PPS H.264 с кодами начала. Отправляется один раз при старте кодера.

### FRAME (0x02) -- Телефон -> Авто

NAL единицы H.264, представляющие видеокадр.

**Параметры кодирования** (устанавливаются сервером VD, настраиваются через handshake):
- Кодек: H.264/AVC
- Профиль: High
- Разрешение: размеры viewport авто (напр., 1806x990)
- Битрейт: 8 Mbps CBR
- Частота кадров: настраивается через `targetFps` в handshake (по умолчанию 30, авто запрашивает 60)
- Интервал IDR: 1 секунда
- SurfaceScaler: периодическая перерисовка каждые `1000/fps` мс обеспечивает вывод кодера на статичном контенте

## Канал данных (0x03)

### NOTIFICATION_POST (0x01) / NOTIFICATION_REMOVE (0x02) -- Телефон -> Авто

Данные уведомления с id, packageName, appName, title, text, timestamp, progressIndeterminate (byte), progress (int32), progressMax (int32). Авто дедуплицирует по ID (обновления заменяют существующие). Касание уведомления запускает приложение-владелец на VD.

### NOTIFICATION_CLEAR (0x04) — Авто → Телефон

Нагрузка: `ClearNotificationMessage` — notification id + package name. Автомобиль закрывает одно уведомление; телефон очищает соответствующее Android-уведомление.

### NOTIFICATION_CLEAR_ALL (0x05) — Авто → Телефон

Пустая нагрузка. Автомобиль закрывает все уведомления; телефон очищает все активные уведомления.

### APP_UNINSTALLED (0x06) — Телефон → Авто

Нагрузка: UTF-8 package name. Телефон подтверждает удаление приложения (в ответ на `APP_UNINSTALL`). Автомобиль удаляет приложение из своей сетки запуска.

### APP_INFO_DATA (0x07) — Телефон → Авто

Нагрузка: `AppInfoDataMessage` — package name, version name, version code, время установки, время обновления, размер приложения (байт). Телефон отправляет метаданные приложения для отображения в диалоге на стороне автомобиля, когда пользователь выбирает "Информация о приложении" из контекстного меню.

### APP_LIST (0x03) -- Телефон -> Авто

Список установленных приложений с packageName, appName, категорией (NAV/MUSIC/COMM/OTHER), iconPng (PNG 96x96).

### CAR_LOG (0x30) -- Авто -> Телефон

Строка текста UTF-8. Авто направляет все логи (включая VideoDecoder и UsbAdbConnection) через этот канал. Телефон пишет в FileLog (`/sdcard/DiLinkAuto/client.log`) с тегом `CarLog`.

### MEDIA_METADATA (0x10) / MEDIA_PLAYBACK_STATE (0x11) -- Телефон -> Авто

Информация о треке и состояние воспроизведения. Пока не заполняется активно.

### MEDIA_ACTION (0x12) -- Авто -> Телефон

Управление медиа (воспроизведение/пауза/следующий/предыдущий). Пока не подключено к MediaSession.

### NAVIGATION_STATE (0x20) -- Телефон -> Авто

Данные состояния навигации. Зарезервировано для интеграции виджета навигации.

## Канал ввода (0x04)

### TOUCH_DOWN (0x01) -- Авто -> Телефон

Событие нажатия одним указателем. Нагрузка — TouchEvent (25 байт):

```
+----------------------+------+
| action                | byte  |  InputMsg.TOUCH_DOWN (0x01)
| pointerId             | int32 |  ID указателя multi-touch
| x                     | float |  нормализовано 0.0-1.0
| y                     | float |  нормализовано 0.0-1.0
| pressure              | float |
| timestamp             | int64 |
+----------------------+------+
```

### TOUCH_MOVE (0x02) -- Авто -> Телефон

Событие перемещения одним указателем. Тот же формат TouchEvent, что и TOUCH_DOWN.

### TOUCH_UP (0x03) -- Авто -> Телефон

Событие отпускания одним указателем. Тот же формат TouchEvent, что и TOUCH_DOWN.

### TOUCH_MOVE_BATCH (0x04) -- Авто -> Телефон

```
+----------------------+------+
| count                 | byte  |  количество указателей
| N × указатель:        |       |
|   pointerId           | int32 |
|   x                   | float |  нормализовано 0.0-1.0
|   y                   | float |  нормализовано 0.0-1.0
|   pressure            | float |
|   timestamp           | int64 |
+----------------------+------+
```

Пакетные события MOVE — все активные указатели в одном сообщении. Уменьшает количество системных вызовов для multi-touch жестов.

### KEY_EVENT (0x10) -- Авто -> Телефон

Событие клавиши (напр., медиа-клавиши, навигационные клавиши). Зарезервировано для будущего использования.

## Константы

```
APP_VERSION_COMPARISON = versionName через semver (с fallback на versionCode для старых авто)
PROTOCOL_VERSION      = 1
CONTROL_PORT          = 9637 (телефон <-> авто, handshake + heartbeat + команды + данные)
VIDEO_PORT            = 9638 (телефон -> авто, только кадры H.264)
INPUT_PORT            = 9639 (авто -> телефон, только события касаний)
VD_SERVER_PORT        = 19637 (телефон <-> сервер VD, только localhost, неблокирующий NIO)
TARGET_FPS            = 60 (настраивается через handshake, по умолчанию 30)
FRAME_INTERVAL_MS     = 1000 / TARGET_FPS (16мс при 60fps, макс. ожидание для циклов видеопути)
HEARTBEAT_INTERVAL    = 3000 мс (только управляющее соединение)
HEARTBEAT_TIMEOUT     = 10000 мс (только управляющее соединение)
MAX_PAYLOAD_SIZE      = 134 217 728 байт (128 МБ)
SERVICE_TYPE (mDNS)   = "_dilinkauto._tcp."
DISPLAY_MODE_MIRROR   = 0
DISPLAY_MODE_VIRTUAL  = 1
```

## Порядок байтов

Все многобайтовые целые числа и числа с плавающей точкой — **big-endian**.
